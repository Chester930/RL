
## 觀察與分析

### rollout_length 排程的效果

```
Step 10k → rollout_length=2：初期 eval 較差（-1571），但方差縮小跡象
Step 20k → rollout_length=3：eval -1442，最低方差（±67），策略短暫穩定
Step 30k → rollout_length=4：eval -1510，長 rollout 帶入更多誤差
Step 40k → rollout_length=5：最不穩定期開始（step 50k 最差 -1605）
Step 60k+：real_buffer 累積足夠多真實資料（60k 筆），模型品質改善
Step 80k：最佳 eval -1323.2，標準差僅 ±24（策略最穩定）
```

**結論**：rollout_length 排程需要 **60k+ 真實步數** 才能讓模型品質趕上 rollout 長度帶來的誤差，建議初始排程放慢（每 20k 步 +1）。

### 為何仍未真正收斂？

```
根本限制 1：Pendulum 動態對模型誤差高度敏感
  sin(θ)、cos(θ)、θ̇ 的非線性動態
  1 步預測誤差在 rollout_length=5 時累積 5 倍

根本限制 2：CPU + 小 ensemble（3 成員）
  論文使用 7 成員 ensemble（GPU），可以更好地估計不確定性
  3 成員的 elite 選擇（n_elite=2）不確定性估計不夠精確

根本限制 3：步數不足（中止於 104k，配置為 150k）
  step 80k 的 -1323.2 顯示仍有改善空間
  預計 150k 完整跑完可能達到 -1200 左右

建議後續方案：
  (A) 完整跑完 150k 步（需不關機，約再 2 小時）
  (B) 改用 HalfCheetah（動態更平滑，更適合 MBPO）
  (C) GPU 上跑 300k 步完整訓練
```

## MBPO 核心原理

```
每個真實步數的訓練迴圈：
  1. 收集 1 個真實轉換（s, a, r, s'）→ 存入 real_buffer
  2. 每 500 步：在 real_buffer 上訓練整合動態模型 p_θ(s'|s,a)
  3. 從 real_buffer 取樣狀態 → 用模型展開 k 步 → 存入 model_buffer
  4. 混合批次（50% real + 50% model）→ 5 次 SAC 梯度更新

核心優勢（相對純 Model-Free SAC）：
  model rollouts 等效於「免費的合成資料」
  → 用少量真實互動訓練高效策略
  → 論文報告樣本效率提升 5-40×（Hopper/Ant 環境）

核心挑戰：
  模型誤差（Model Bias）會累積進 Q 估計
  → MBPO 使用短 rollout 排程 + 整合不確定性估計緩解
  → CPU demo 仍受 ensemble 大小與訓練步數限制
```
