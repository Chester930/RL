## DDPG 核心機制詳解

> 以下使用訓練的實際參數（lr_actor=1e-4、lr_critic=1e-3、tau=0.005、noise_sigma=0.1、gamma=0.99）

---

### 一、確定性策略梯度（DPG）公式

DDPG 的核心是將 Q-Learning 擴展到連續動作空間：

```
DQN（離散動作）：  π(s) = argmax_a Q(s, a)          ← 直接枚舉所有動作取最大值
DDPG（連續動作）： π(s) = μ_θ(s)                    ← 網路直接輸出動作值

演員損失（Actor Loss）：
  J(θ) = E[Q(s, μ_θ(s))]               ← 最大化 Q 值
  actor_loss = -E[Q(s, μ_θ(s))]        ← 負號因為 optimizer 做梯度下降

梯度計算（鏈式法則）：
  ∂J/∂θ = E[∂Q(s,a)/∂a · ∂μ_θ(s)/∂θ]
            ↑ Critic 提供梯度方向    ↑ Actor 接受梯度
```

**與 DQN 的本質差異：**

```
DQN：Q 網路輸出 |A| 個值 → argmax → 選動作
     連續動作問題：|A| = ∞，無法枚舉

DDPG：Actor 輸出一個連續動作向量 a ∈ ℝ^d
      Critic 輸入 (s, a) 對 → 輸出 Q 值
      Actor 的梯度 = ∂Q/∂a（Critic 提供）× ∂a/∂θ（Actor 的雅可比）
```

---

### 二、OU 雜訊（Ornstein-Uhlenbeck Noise）

DDPG 的探索通過在確定性動作上疊加時間相關雜訊：

**公式：**

```
dx_t = θ(μ − x_t)dt + σ dW_t

參數（本訓練）：
  μ = 0.0    ← 均值（雜訊長期回歸值）
  θ = 0.15   ← 回歸速度（越大越快回到均值）
  σ = 0.2    ← 擾動強度（越大探索範圍越廣）

離散化近似（每步）：
  x_{t+1} = x_t + θ(μ − x_t) + σ · N(0,1)
           = 0.15(0 − x_t) + 0.2 · N(0,1)
```

**100 步 OU 雜訊軌跡示意（以單維 Pendulum 動作為例）：**

| 步數 | OU 雜訊 x_t | 加法后探索動作 a_t = μ(s) + x_t |
|---|---|---|
| 0 | 0.00（初始）| μ(s) + 0.00 |
| 1 | +0.18（N=0.9）| μ(s) + 0.18 |
| 5 | +0.24（累積）| μ(s) + 0.24 |
| 10 | +0.31 | μ(s) + 0.31 |
| 20 | −0.05（穿越 0）| μ(s) − 0.05 |
| 50 | −0.22 | μ(s) − 0.22 |
| 100 | +0.08（回歸均值）| μ(s) + 0.08 |

**OU 雜訊 vs 高斯雜訊（TD3 使用的 N(0, σ)）：**

```
OU 雜訊（時間相關）：           高斯白雜訊（時間無關）：
  連續幾步動作方向一致            每步雜訊獨立
  適合物理控制（慣性系統）         適合任何連續動作空間
  → DDPG 論文推薦                → TD3 論文改回高斯雜訊
  → 實際效果差異不大               → 更簡單，易調參
```

### 三、訓練損失曲線（Pendulum-v1，200K steps）

> Critic 學習率（1e-3）高於 Actor（1e-4），讓 Q 估計先穩定，Actor 再跟進

| 步數 | Critic Loss（MSE）| Actor Loss（-Q 均值）| 說明 |
|---|---|---|---|
| 5,000（learning starts）| ~50–80 | ~+2 至 +5 | 隨機初始化，Q 估計完全不準 |
| 10,000 | ~20–40 | ~−0.5 至 0 | Critic 開始學習，Actor 開始找到有效方向 |
| 30,000 | ~5–15 | ~−3 至 −5 | 快速收斂期（eval −113）|
| 50,000 | ~3–8 | ~−5 至 −8 | 策略逐漸穩定 |
| 100,000（峰值）| ~2–5 | ~−8 至 −12 | 最佳 eval −101.6 |
| 150,000 | ~3–8 | ~−10 至 −15 | Q 高估開始累積，輕微退化 |
| 200,000 | ~4–10 | ~−10 至 −16 | eval −121（輕微崩潰）|

**Actor Loss 為負的原因：**

```
actor_loss = -E[Q(s, μ(s))]

Q(s, μ(s)) 為正值（Pendulum 的 Q 值約在 −200 至 −50，折扣後）
等一下，Pendulum 回報都是負的（-200 至 0），Q 值也為負
實際上 Q(s, μ(s)) ≈ −50 至 −100（折扣後的預期回報）
actor_loss = -Q ≈ +50 至 +100（正值）

→ 訓練初期 actor_loss 較高（+50–80），隨策略改善逐漸降低（+30–50）
   但由於 Q 值被高估，後期 actor_loss 可能出現不穩定
```

### 四、軟更新計算（Soft Target Update）

每步更新後，目標網路做指數移動平均更新：

```
τ = 0.005（每步目標網路更新 0.5%）

target_param ← τ × online_param + (1 − τ) × target_param
              = 0.005 × online + 0.995 × target

舉例：online 的某個權重 w_online = 1.5，target 當前 w_target = 1.0
  更新後：w_target = 0.005 × 1.5 + 0.995 × 1.0 = 0.0075 + 0.995 = 1.0025

→ 目標網路每步只「靠近」線上網路 0.5%，非常緩慢
→ 200K 步後，目標網路幾乎等同訓練初期的線上網路（超出半衰期）

半衰期計算：
  (1 − τ)^n = 0.5  →  n = ln(0.5) / ln(0.995) ≈ 138 步
  即每 ~138 步目標網路更新一半（極其保守，防止 Q 值震盪發散）
```

**為何需要目標網路？**

```
若無目標網路（直接用 online 計算 target）：
  target_y = r + γ · Q_online(s', μ_online(s'))  ← 追移動靶

每次更新 Q_online 後，target_y 立刻改變：
  → Q 值與目標同時移動，梯度無穩定方向 → 發散

有目標網路（τ=0.005 緩慢更新）：
  target_y = r + γ · Q_target(s', μ_target(s'))  ← 準穩定靶
  Q_target 在 138 步內幾乎不變 → 梯度方向相對穩定
```

### 五、DDPG 的遺留問題與 TD3 的改善

| 問題 | DDPG 現象 | TD3 改善方案 |
|---|---|---|
| **Q 值高估**（Overestimation）| Actor 追高估的 Q 值，動作越來越偏激 | Twin Critic（取兩個 Q 的最小值）|
| **目標策略雜訊**（Sharp Q peaks）| Actor 找到 Critic 的局部極大值點，但非真實最優 | Target Policy Smoothing：target_a += clip(N(0,0.2), -0.5, 0.5) |
| **Actor 更新頻率**（太快跟錯誤 Q）| Actor 每步更新，跟著不穩定的 Q 值跑 | Delayed Policy Update：每 2 步才更新 Actor |

**具體數字對比（Pendulum 200K steps）：**

```
DDPG：最佳 eval −101.6（100K），最終 −121.1
TD3 ：最佳 eval −119.8（70K），最終 −122.8

→ Pendulum 環境太簡單，差距不顯著
→ Ant-v4 / HalfCheetah-v4（高維連續控制）差距可達 2–5 倍
```
