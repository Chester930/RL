# RL² 訓練日誌

> 環境：5-Armed Bernoulli Bandit（每任務隨機採樣臂機率）  
> Meta 設定：每任務 50 步，隨機基線命中率：1/5 = 0.200（完全隨機選臂）

---

## 演算法特有指標

| 指標 | 說明 |
|---|---|
| 最優臂命中率（全段） | 整個任務內選到最佳臂的比例 |
| 最優臂命中率（後半段） | 步數 ≥ 25 後的命中率（排除探索期，衡量剝削能力） |
| `entropy` | GRU 策略的熵（越低表示策略越確定） |
| `approx_kl` | PPO 近似 KL 散度 |

## 學習曲線觀察

### 探索轉剝削的轉換點

GRU 熵在 Run 2 從 1.003（update 1100）持續下降至 0.227（update 2000）。
update 1200 時熵為 0.848，處於「有足夠確定性但保留探索彈性」的最佳區間，此時後半段命中率達到峰值 0.546。

### 過度剝削問題（update 1200 後）

update 1200 之後熵繼續崩潰（0.848 → 0.227），策略過早鎖定某些臂。
由於每個 bandit 任務的臂機率完全隨機，過度確定的策略反而無法適應任務差異，導致命中率下降至接近隨機基線（update 1700：0.222）。

update 1800 的短暫回升（0.498）顯示 GRU 仍具備學習能力，但整體趨勢向下。

### GRU 是否學到「試一下、記結果、鎖定最佳臂」的行為

有學到。後半段命中率峰值 0.546 vs 隨機 0.200，提升 **2.73 倍**，代表 GRU hidden state 確實在跨 step 累積探索資訊並做出較好決策。

最終熵 0.227 顯示策略高度確定化，但確定化過早導致通用性下降。

## 教學重點

RL² 的關鍵演示：同一個 GRU 在從未見過的新任務上，
能在前幾步探索後自動切換到剝削——這正是「學習如何學習」的體現。

```
前 25 步（探索期）：GRU 不確定哪臂最好，嘗試不同臂
後 25 步（剝削期）：GRU 鎖定認為最好的臂，命中率顯著提升

GRU 輸入：[prev_action_onehot(5) | prev_reward(1) | prev_done(1)] = 7 維
GRU 輸出 → actor head → 臂機率分布
           → critic head → 狀態價值（PPO 用）
```

| | 目標 | 實際（最佳）| 說明 |
|---|---|---|---|
| 後半段命中率 | ≥ 0.600 | **0.546**（update 1200）| 達目標的 91% |
| vs 隨機基線 | — | **2.73×** | 顯著超過隨機 |

## 後續改進方向

若要進一步提升：
1. **早停策略**：在 update 1200 附近停止，避免 entropy 崩潰
2. **entropy 下界約束**：設定 entropy ≥ 0.3 的 KL 懲罰，防止過度剝削
3. **更長任務**（steps_per_task 50→100）：給 GRU 更多時間識別最優臂
4. **更多臂**（5→10 armed）：增加任務難度，迫使策略保持探索
