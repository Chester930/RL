# Behavioral Cloning 訓練日誌

環境：Pendulum-v1（單關節扭矩控制）｜Expert：SAC（pre-trained）
示範資料：50 集 × 200 步 = 10,000 transitions｜BC 訓練：100 epochs

---

## 核心概念

### 什麼是 Behavioral Cloning？

```
收集專家示範資料（state, action 配對）
→ 監督學習：讓 policy 的輸出盡量接近專家動作
→ loss = MSE(policy(state), expert_action)
```

BC 完全繞開 RL 最難的兩個問題：獎勵設計與探索。
直覺、易實作，是機器人學習的起點。

### 為什麼 BC 有根本限制？

```
訓練時只看到「好的狀態」（專家軌跡附近）
→ 測試時稍微偏離 → policy 不知道怎麼辦 → 錯誤累積 → 崩潰
```

這個現象叫做 **Distribution Shift（分佈偏移）**。

---

## Step 1｜收集 SAC Expert 示範資料

| 指標 | 數值 |
|:---|:---|
| Expert 來源 | SAC（pre-trained on Pendulum） |
| 收集集數 | 50 集 |
| 總 transitions | 10,000 筆 |
| Expert eval 均值 | -170.3 ± 85.6 |
| State shape | (10000, 3)：[cos θ, sin θ, θ̇] |
| Action shape | (10000, 1)：扭矩 ∈ [-2, 2] |

> 注意：SAC expert 在 Pendulum 的 eval = -170，並非最優（DDPG 可達 -101）。
> 這是因為 SAC 的 entropy 正則化在極簡任務中過度探索。
> 但其示範資料足以讓 BC 學會擺錘直立的基本動作。

---

## Step 2｜BC 監督學習訓練

| 設定 | 數值 |
|:---|:---|
| 網路架構 | MLP：3 → 256 → 256 → 1（tanh 輸出） |
| 損失函式 | MSE(policy(s), expert_action) |
| 訓練 epochs | 100 |
| 學習率 | 1e-3 |

### 訓練曲線

| Epoch | BC Loss | Eval Return |
|:---:|:---:|:---:|
| 1 | 0.3687 | -1336.0 ± 46.1 |
| 10 | 0.1309 | -575.0 ± 306.0 |
| 20 | 0.1126 | -178.6 ± 72.6 |
| 30 | 0.0859 | -204.1 ± 90.0 |
| 40 | 0.0722 | -183.8 ± 126.7 |
| 50 | 0.0641 | -134.0 ± 61.4 |
| 60 | 0.0568 | -203.1 ± 103.4 |
| 70 | 0.0633 | -159.1 ± 52.6 |
| 80 | 0.0507 | -190.0 ± 102.5 |
| 90 | 0.0427 | -186.3 ± 89.4 |
| **100** | **0.0420** | **-144.6 ± 67.9** |

**觀察**：
- Loss 穩定下降（0.37 → 0.04），監督學習本身收斂良好
- Eval 在 -134 ～ -203 間震盪，最終約 -145
- 高標準差（±67）：BC policy 在某些初始狀態可行，在其他狀態崩潰

## Step 3｜分佈偏移測試（Distribution Shift Demo）

**測試設計**：固定初始角度從 0°（直立）到 180°（朝下），
比較 BC Policy 與 SAC Expert 的表現。

BC 的 demo 資料 80%+ 都是 θ ≈ 0 附近的狀態（SAC 快速讓擺錘直立）。
角度越大 → BC 越少見此狀態 → 越可能崩潰。

### 結果對比

| 初始角度 | SAC Expert | BC Policy | 狀態 |
|:---:|:---:|:---:|:---:|
| 0° | -0.0 | -0.0 | ✅ 分佈內，完美 |
| 30° | -132.2 | -132.7 | ✅ 分佈內，相近 |
| 60° | -127.1 | -127.3 | ✅ 分佈內，相近 |
| 90° | -123.4 | -122.2 | ✅ 邊界，勉強 |
| **120°** | **-122.2** | **-245.0** | ❌ 分佈外，崩潰 |
| 150° | -231.1 | -231.3 | ⚠ 偶然相近 |
| **180°** | **-353.5** | **-436.5** | ❌ 分佈外，崩潰 |

> Pendulum 回報越靠近 0 越好（最優 ≈ 0，最差 ≈ -1600）。

### 結論

- BC **在訓練分佈內**（θ 接近 0°）表現接近 SAC Expert
- BC **在分佈外**（θ ≥ 120°）效能顯著下降，崩潰幅度達 2 倍
- SAC 在所有角度穩定（off-policy 探索覆蓋各種初始條件）

**課堂教學點**：
「BC 不是毫無用處——它在見過的狀態附近表現合理。
 關鍵問題是：真實機器人部署時，初始狀態千變萬化，
 BC 碰到略微不同的情況就會累積誤差直到崩潰。
 這就是為什麼我們需要讓 agent **自己探索、從錯誤中修正**——這就是 RL。」

## 與其他演算法的關係

| 演算法 | Pendulum 最佳 Eval | 訓練方式 |
|:---|:---:|:---|
| **BC** | **-144.6**（高方差）| 監督學習，無探索 |
| SAC Expert | -170.3 | off-policy RL，entropy 正則化 |
| DDPG | -101.6 | off-policy，deterministic |
| TD3 | -119.8 | off-policy，twin critic |

BC 在分佈內可與 SAC Expert 匹敵，但泛化能力遠遜於任何 RL 演算法。
