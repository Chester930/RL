# Dueling DQN | 競爭 DQN

## 論文

Wang, Z., Schaul, T., Hessel, M., van Hasselt, H., Lanctot, M., & de Freitas, N. (2016).  
*Dueling Network Architectures for Deep Reinforcement Learning*. ICML 2016. arXiv:1511.06581.

---

## 核心思想 (Key Idea)

Dueling 架構將 Q(s, a) 分解為兩個元件：
- **V(s)**: 狀態價值 (State value) — 在不考慮動作的情況下，這個狀態有多好？
- **A(s, a)**: 優勢 (Advantage) — 相對於平均水平，動作 a 有多好？

```
Q(s, a) = V(s) + A(s, a) - mean_{a'} A(s, a')
```

減去平均優勢可確保分解過程具有**可識別性 (Identifiability)**，否則 V 和 A 會因為有無數種組合而無法唯一確定。

**白話解釋：**

原本的 DQN 直接學 Q(s, a)——「在這個狀態下選這個動作，最終能活多久」。這個數字同時包含了兩件事：

```
① 這個狀態本身有多好？（不管選哪個動作）
② 這個動作在這個狀態下比其他動作好多少？
```

Dueling 把這兩件事拆開來分別學：

```
V(s)    = 這個狀態本身的分數
A(s, a) = 「選動作 a」比「選平均動作」好多少分

Q(s, a) = V(s) + A(s, a) - mean(A)
         ↑ 狀態分   ↑ 動作額外貢獻   ↑ 平均修正
```

**具體 CartPole 例子：**

桿子幾乎直立、小車在中央（理想狀態）：
```
V(s)    = 99    ← 這個狀態本身很好，不管推哪邊都有機會活很久
A(←)    = +0.3  ← 往左稍微好一點
A(→)    = -0.3  ← 往右稍微差一點
mean(A) = 0.0   ← 平均優勢

Q(←) = 99 + 0.3 - 0.0 = 99.3
Q(→) = 99 + (-0.3) - 0.0 = 98.7
```

桿子大幅傾斜（危險狀態）：
```
V(s)    = 20    ← 這個狀態本身很危險
A(←)    = +8    ← 往左（正確方向）好很多
A(→)    = -8    ← 往右會馬上倒
mean(A) = 0.0

Q(←) = 20 + 8 - 0.0 = 28   ← 還有救
Q(→) = 20 + (-8) - 0.0 = 12 ← 幾乎沒救
```

## 為什麼有效 (Why It Works)

在許多狀態下，**採取哪種動作並不重要**。例如在迷宮中如果你遠離牆壁，所有方向的動作價值都差不多。價值流 (Value stream) 可以專注於學習 V(s) 而不被動作價值的微小差異幹擾，進而達到更好的泛化能力。

對於真正關鍵的轉折點，優勢流 (Advantage stream) 則能精確地捕捉不同動作間的相對優劣。

**白話解釋：**

在普通 DQN 裡，每次你選了「往左」這個動作，網路只更新 Q(s, 左)。Q(s, 右) 這一步完全沒更新，要等下次真的選「往右」才會學到。

在 Dueling DQN 裡，每次更新任何動作，V(s) 都會一起更新——因為 V(s) 被兩個 Q 值共用：

```
普通 DQN：選了「往左」→ 只更新 Q(s, 左)，Q(s, 右) 沒動

Dueling：  選了「往左」→ 更新 A(s, 左)，也更新 V(s)
                         V(s) 更準了 → Q(s, 右) 也自動變準了！
```

類比：你考了一次數學選擇題，選了「A」。
- 普通 DQN：只知道「選A對不對」這件事。
- Dueling：除了知道「A對不對」，還同時更新了「這道題的難度」（V(s)）——下次看到類似題，你對B、C、D選項的判斷也變準了。

這讓 Dueling 在動作數量多、或很多狀態下動作差異小的環境特別有效。

## 網路架構 (Architecture)

```
狀態 ->  共享骨幹網路 (CNN 或 MLP)
                |
         ┌─────┴──────┐
     價值流 (Value)   優勢流 (Advantage)
     FC -> V(s)     FC -> A(s, a_1..n)
                |
         Q = V + A - mean(A)
```

**白話解釋：**

```
輸入（4個數字：位置、速度、角度、角速度）
  │
  ▼
共享骨幹（MLP，把 4 個數字轉換成 256 個特徵）
  │
  ├──▶ 價值流：256 → 1 個數字（V(s)，這個狀態值多少）
  │
  └──▶ 優勢流：256 → 2 個數字（A(←) 和 A(→)，兩個動作各自的優勢）
  
最後合併：
  Q(←) = V + A(←) - [A(←) + A(→)] / 2
  Q(→) = V + A(→) - [A(←) + A(→)] / 2
```

跟普通 DQN 唯一的差別，就是把最後一層從「直接輸出 Q」改成「先分開輸出 V 和 A，最後再合併」。更新公式、Replay Buffer、Target Network 完全不變。

## 可識別性 (Identifiability)

若沒有額外的約束，V 和 A 可以是任意值（例如 V=0 且 A=Q，或是 V=Q 且 A=0 都能滿足 Q=V+A）。實務上常用的約束有：

1. **減去平均值 (Mean subtraction)**（本實作採用）：`Q = V + A - mean(A)`
2. **減去最大值 (Max subtraction)**：`Q = V + A - max(A)`

減去平均值通常表現較穩健，因為它具有較好的梯度傳播特性。

**白話解釋（這是最難懂的部分）：**

假設最終 Q(←) = 100，Q(→) = 99。如果只說 Q = V + A，有無數種拆法都成立：

```
拆法 A：V = 0，   A(←) = 100，A(→) = 99   ← V 完全沒意義
拆法 B：V = 50，  A(←) = 50，  A(→) = 49
拆法 C：V = 100， A(←) = 0，   A(→) = -1   ← A 才有意義
```

網路如果可以隨意選任一種，它就不知道 V 和 A 各自應該是多少，學習會很混亂。

**「減去平均值」強制讓 A 的均值 = 0**：

```
A(←) + A(→) 的平均必須 = 0
→ 只有拆法 C 的感覺才對（A 的平均是 0，V 承擔主要數值）

實際上：mean(A) = [100 + 99] / 2 = 99.5
→ Q(←) = V + A(←) - 99.5 ... 這樣 A 就被「歸零」了
```

這樣 V(s) 就真的代表「這個狀態本身的價值」，A(s,a) 就真的代表「動作的相對優勢」，兩者各司其職，不會互相干擾。

## 什麼時候最有效 (When Dueling Helps Most)

- 許多動作具有相近價值的狀態（優勢 ≈ 0）。
- 包含許多無關緊要動作的遊戲。
- 動作空間巨大且個別動作對環境影響較小的場景。

**白話解釋（以 CartPole 為例）：**

CartPole 的很多狀態下，「往左推一點點」和「往右推一點點」的結果其實差不多——桿子幾乎直立時，推哪邊都能活很久。

```
桿子直立（θ ≈ 0）：
  Q(←) ≈ 99.3，Q(→) ≈ 98.7   ← 差 0.6，選哪個都差不多

桿子大幅傾斜（θ = 0.15）：
  Q(←) ≈ 28，Q(→) ≈ 12       ← 差 16，這時選哪個差很多
```

普通 DQN 要把 Q(←) 和 Q(→) 各自學準，才能正確判斷。
Dueling DQN 先學好「這個狀態值多少（V）」，再學「哪個動作比較好（A）」——在差不多的狀態，A 自然學到接近 0，V 學到精準的狀態估計。

效果最差的場景：每個動作都差很多，A 的值域很大，V 和 A 都要很精準——此時 Dueling 相對優勢不明顯。

## 關鍵公式 (Key Equations)

```
Q(s, a; theta, alpha, beta) = V(s; theta, beta) + [A(s, a; theta, alpha) - (1/|A|) sum_{a'} A(s, a'; theta, alpha)]
```

其中 theta = 共享引數，alpha = 優勢流引數，beta = 價值流引數。

**白話解釋：**

```
theta  = 共享骨幹的引數（輸入層 → 256維特徵）
beta   = 價值流的引數（256維 → V(s) 這一條路的權重）
alpha  = 優勢流的引數（256維 → A(s,a) 這一條路的權重）

(1/|A|) sum A  =  mean(A)  =  把所有動作的優勢取平均
```

整句話翻譯：**Q 值 = 用共享骨幹+價值流算出的狀態分數，加上用共享骨幹+優勢流算出的動作額外貢獻，再減去平均優勢（讓分解唯一）**。

## 與普通 DQN 的唯一差異

Dueling DQN 相對於 DQN，只改了一件事：**網路架構的最後幾層**。

```
DQN：       特徵 → 線性層 → Q(←), Q(→)            （直接輸出）

Dueling：   特徵 ┬→ 線性層 → V(s)                  （價值流）
                 └→ 線性層 → A(←), A(→)            （優勢流）
                         ↓ 合併
                    Q = V + A - mean(A)
```

所有其他部分完全不變：
- Replay Buffer：一樣
- Target Network：一樣
- ε-greedy 探索：一樣
- 損失函式（Huber loss）：一樣
- Double DQN 目標計算：一樣（本實作同時使用）
