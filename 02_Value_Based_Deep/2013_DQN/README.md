# DQN（深度 Q 網路）

> 年代：2013/2015｜方法：無模型、異策略、步數式線上更新｜環境：CartPole-v1

---

## 1. DQN 是什麼？

Q-Learning 能直接學出最優策略，但有一個致命弱點：**Q 表大小 = 狀態數 × 動作數**。FrozenLake 只有 16 個格子，CartPole 的狀態卻是四個連續數值（位置、速度、角度、角速度）的組合——根本沒辦法用表格存。

**DQN 的做法**：把 Q 表換成神經網路。網路輸入狀態 s，輸出所有動作的 Q 值：

```
Q_theta(s, a)   ←   神經網路近似
取代
Q[state, action]  ←   表格查詢
```

這個想法早在 1990 年代就有人試過，但直接用 Q-Learning 更新神經網路會遇到兩個問題，導致訓練不穩定甚至發散。DeepMind 2013/2015 年的突破，正是給出了這兩個問題的解法。

## 2. CartPole-v1 環境

### 2-1. 場景描述

一臺小車在水平軌道上左右滑動，車上有一根用鉸鏈連線的桿子。桿子靠重力自然會往旁邊倒，**你每步只能選擇「推左」或「推右」，讓小車移動，靠慣性把桿子撐住**。

```
                  桿子（用鉸鏈固定，可自由旋轉）
                    │
                    │  ← θ（角度，直立時 = 0）
                    │
               ┌────┴────┐
               │  小 車  │  ← x（位置，中央 = 0）
               └─────────┘
    ←──────────────────────────────→
    軌道（x = −4.8 ～ +4.8，超出即結束）
    ← 推左                     推右 →
```

---

### 2-2. 四個觀測值的物理意義

FrozenLake 的狀態是「在哪個格子」（1 個整數），CartPole 的狀態是**描述這個物理系統當下的四個連續數值**：

```
s = [ x,  ẋ,  θ,  θ̇ ]
      ①   ②   ③   ④
```

| # | 符號 | 意義 | 範圍 | 直覺 |
|:---:|:---:|:---|:---:|:---|
| ① | x | 小車在軌道上的位置 | −4.8 ～ +4.8 | 0 = 正中央；超出邊界就結束 |
| ② | ẋ | 小車的移動速度 | 無限制 | 正 = 往右跑；負 = 往左跑 |
| ③ | θ | 桿子相對於垂直線的角度（弧度）| −0.42 ～ +0.42 | 0 = 直立；正 = 往右傾；負 = 往左傾 |
| ④ | θ̇ | 桿子倒下的角速度 | 無限制 | 正 = 往右加速傾倒；負 = 往左加速傾倒 |

**終止條件**（任一成立就結束這集）：
- `|θ| > 0.209`（約 12°）：桿子倒太多
- `|x| > 2.4`：小車跑出軌道
- 撐過 500 步：任務成功

---

### 2-3. 三種典型狀態

```
  ① 正常（直立）     ② 往右傾（危險）     ③ 往左跑出界（失敗）

        │                    /                   │
        │                   /  ← θ > 0           │
        │                  /                     │
   ┌────┴────┐       ┌────/────┐           ┌────┴────┐
   │  小 車  │       │  小 車  │           │  小 車  │ → 出界
   └─────────┘       └─────────┘           └─────────┘
   ──────────────    ──────────────    ──────────────────────
   x≈0, θ≈0         x≈0, θ>0          x < −2.4（結束）
   → 安全            → 需往右推         → 已終止
```

---

### 2-4. 控制直覺

**為什麼桿子會倒？**  
鉸鏈連線讓桿子可以自由旋轉，重力會讓它持續傾倒。一旦偏離垂直，不做任何動作就會越倒越快。

**怎麼靠「推小車」撐住桿子？**  
```
桿子往右傾（θ > 0）→ 把小車往右推 → 小車向右加速
→ 慣性讓桿子相對往左擺 → θ 縮小 → 桿子回正
```

這和你走路時靠移動腳步來保持平衡是同樣的原理。

**為什麼難？**  
- 四個狀態連續且相互影響（推太猛反而更難控制）
- 只靠「+1 活著、0 倒下」的稀疏獎勵學習，沒有任何「快倒了請注意」的提示
- 表格型 Q-Learning 根本存不下這個連續狀態空間（需要神經網路）

---

### 2-5. 觀測值摘要

```
觀測值（連續 4 維）：
  s[0]  x     ：小車位置（−4.8 ～ +4.8，超出結束）
  s[1]  ẋ     ：小車速度（正 = 往右）
  s[2]  θ     ：桿子角度（弧度，正 = 往右傾，|θ|>0.209 倒下）
  s[3]  θ̇     ：桿子角速度（正 = 往右加速傾倒）

動作（離散 2 維）：
  0 = 向左推力
  1 = 向右推力

獎勵：每活過一步 r = +1（沒有方向提示，純粹靠存活時間）
終止：|θ| > 0.209（約 12°）或 |x| > 2.4，或撐過 500 步
目標：讓桿子不倒，盡量活到 500 步（回報 = 存活步數）
```

## 3. 兩大核心問題

### 問題一：樣本相關性（Sample Correlation）

Q-Learning 做線上更新時，連續的樣本 `(s_t, a_t, r_t, s_{t+1})` 在時間上高度相關——每一步都是上一步的直接延續。這違反了監督式學習對「訓練樣本獨立同分佈」的假設，導致梯度更新方向劇烈震盪，網路難以收斂。

### 問題二：移動目標（Moving Target）

Q-Learning 的更新目標是：

```
y = r + γ × max_{a'} Q(s', a')
```

但 Q 值本身就是我們正在訓練的網路輸出。每次更新後，網路引數 θ 改變，下一步的目標 y 也隨之改變——就像在追一個會跑的標靶，很容易陷入震盪或發散。

---

## 4. 兩大解決方案

### 解法一：經驗回放（Experience Replay）

不在每步立刻更新，而是把每次轉移 `(s, a, r, s', done)` 存入一個大型**回放緩衝區（Replay Buffer）**。每次更新時，**隨機抽取一個迷你批次**來計算梯度：

```
緩衝區容量：50,000 筆
每次更新：隨機抽 64 筆
```

**為什麼有效**：隨機抽樣打破了時間相關性，每批次的樣本來自不同時間點，分佈接近獨立同分佈，讓梯度更新更穩定。

**額外好處**：同一筆轉移可以被多次抽到，提升樣本效率。

### 解法二：目標網路（Target Network）

維護兩個結構相同的網路：

```
線上網路 Q_theta     ← 每步更新，用來選動作和計算當前 Q
目標網路 Q_theta-    ← 定期同步，只用來計算 TD 目標 y
```

每隔 C 步（此實作為 500 步），把線上網路的引數複製給目標網路：

```
每 500 步：Q_theta-  ←  Q_theta
```

**為什麼有效**：TD 目標 `y = r + γ × max Q_theta-(s')` 在 C 步之內是固定的，訓練不再追著移動的標靶跑，收斂大幅穩定。

## 5. DQN 演演算法步驟

```
初始化：線上網路 Q_theta，目標網路 Q_theta-（= Q_theta），回放緩衝區 D

重複以下流程（共 200,000 步）：

步驟一：選動作（ε-greedy）
         以機率 ε  → 隨機選動作（探索）
         以機率 1-ε → 選 Q_theta(s) 最大的動作（利用）
         ε 從 1.0 線性衰減到 0.01（前 50,000 步）

步驟二：執行動作，觀察 (r, s', done)

步驟三：存入緩衝區
         D.push(s, a, r, s', done)

步驟四：等緩衝區累積足夠資料（≥ 1,000 筆）後開始更新
         從 D 隨機抽 64 筆迷你批次

步驟五：計算 TD 目標（用目標網路）
         y_i = r_i                                            （若 done）
         y_i = r_i + γ × max_{a'} Q_theta-(s'_i, a')        （否則）

步驟六：計算損失並更新線上網路
         L = (1/64) × Σ Huber(y_i − Q_theta(s_i, a_i))
         theta ← theta − α × ∇L

步驟七：每 500 步同步目標網路
         Q_theta- ← Q_theta
```

### 步驟五到七的白話解釋

**步驟五：算「這次應該得幾分」（正確答案）**

網路的工作是預測「選這個動作，最終能活多久」。但我們不知道真正的答案，只能用下一刻的情況來估計：

```
如果桿子已經倒了（done = True）：
    正確答案 = 這步的獎勵 r（沒有未來了，就這樣）

如果還在繼續（done = False）：
    正確答案 = 這步的獎勵 r + 0.99 × （下個狀態最好能拿到多少）
                                         ↑
                              由「目標網路」來估計這個數字
```

比喻：你在打遊戲，剛走了一步。「這步值多少分」不只看這步得了幾分，還要看「這步把你帶到了多好的局面」。如果這步讓你進了決勝局，那分數就很高。

---

**步驟六：比較「預測」和「正確答案」，然後修正網路**

線上網路現在猜了一個值 `Q_theta(s, a)`，而我們算出了「正確答案」`y`。兩者之差就是「這次猜錯了多少」。

```
差距（誤差）= y（正確答案）− Q_theta(s, a)（網路的預測）

差距大 → 網路猜很錯 → 大幅調整引數
差距小 → 網路猜接近 → 小幅微調
```

比喻：就像做完一套數學題，對答案，把答錯的題目找出來，重新把那種型別練熟。每次練（更新）都讓網路在這類狀況的預測更準一點。

`ΔL` 就是告訴網路「往哪個方向改引數，才能讓誤差變小」（這是微積分的梯度下降，你只需要知道「它讓誤差越來越小」就夠了）。

---

**步驟七：每 500 步「更新答案本」**

為什麼不用同一個網路當正確答案？

問題：如果你今天答錯了一題，改完之後，「正確答案」也跟著變了——你等於在追一個一直在跑的標靶，永遠追不上。

解法：目標網路就像「凍結的教科書」。每 500 步才更新一次。這 500 步之內，「正確答案」保持不變，讓訓練有一個穩定的目標可以收斂。

```
線上網路：每步都在學習、改變（活的引數）
目標網路：定期從線上複製，其他時間凍結（穩定的參照）
```

## 6. 步數式訓練 vs 集數式訓練

| 比較專案 | REINFORCE / SARSA / MC | DQN |
|:---|:---|:---|
| 更新單位 | 每集結束後 | 每步（只要緩衝區夠大）|
| 訓練計數單位 | 集數 | 步數 |
| 需要整集? | MC 需要，SARSA 不需要 | 不需要（步數到就更新）|
| 樣本複用 | 否（每筆只用一次）| 是（同一筆可被多次抽到）|

DQN 以步數為單位訓練的原因：它不需要等整集結束就能更新，且緩衝區裡的舊資料仍然有效（異策略），讓訓練更有彈性。

**白話解釋：**

以前的演算法（REINFORCE）要等桿子倒下、整集結束，才能回頭更新一次。就像打完一整局棋才能覆盤。

DQN 不需要等。只要緩衝區裡有 1,000 筆資料，每走一步就馬上抽 64 筆出來學一次。而且舊資料不會丟掉，五分鐘前發生的事情這步還可以拿來學——這是 Replay Buffer 帶來的額外好處。

```
REINFORCE：走完一局（500步）→ 學一次  →  每局只能學 1 次
DQN：      每走 1 步           → 學一次  →  500步能學 500 次
```

這就是為什麼 DQN 的進步速度快很多。

## 7. Huber 損失 vs MSE

```
MSE：   L = (y - Q)^2                    對異常值敏感
Huber： L = (y - Q)^2 / 2        （|y-Q| ≤ 1）
        L = |y - Q| - 1/2         （|y-Q| > 1）
```

Huber 損失在誤差小時等同 MSE（平滑梯度），在誤差大時退化為 L1（有界梯度）。這讓訓練初期梯度不會因為 Q 值估計不準而爆炸。

**白話解釋：**

損失函式的作用是「量化答錯了多少」，然後用這個數字來決定「應該調整引數多少幅度」。

MSE（均方誤差）的問題：

```
誤差 = 2  →  更新幅度 = 4   （2的平方）
誤差 = 10 →  更新幅度 = 100  （10的平方）
```

訓練初期 Q 值完全不準，誤差很容易到 50、100。平方之後更新幅度變成 2500、10000——網路被修改得太猛，直接爆掉。

Huber 損失的做法：誤差小時用平方（精確），誤差大時改用「線性」，設個上限：

```
誤差 = 2  →  更新幅度 ≈ 2    （小誤差，精細調整）
誤差 = 10 →  更新幅度 ≈ 9.5  （大誤差，有上限，不爆炸）
```

類比：考試改分時，一般每錯一分扣一分（線性）。但如果有人某題寫出天馬行空的答案，老師不會給他扣 100 分，頂多扣這題的滿分——這就是 Huber 損失的概念。

## 9. 訓練結果

訓練 200,000 步（CartPole-v1）的典型進度：

| 步數 | eval 回報 | ε | loss | 說明 |
|:---:|:---:|:---:|:---:|:---|
| 10,000 | ~30 | 0.82 | 高 | 仍在大量探索，Q 值剛開始學習 |
| 50,000 | ~150 | 0.20 | 中 | ε 接近終點，策略開始穩定 |
| 100,000 | ~350 | 0.01 | 低 | Q 值收斂，策略接近最優 |
| 200,000 | ~490 | 0.01 | 極低 | 穩定解決，接近滿分 500 |

**為什麼 DQN 能穩定到 ~490，而 REINFORCE 只有 ~25？**  
REINFORCE 的高方差 + 沒有 replay 讓訓練不穩定；DQN 的 replay + target network 讓損失穩定下降，策略能持續改善。

## 10. DQN vs Q-Learning 比較

| 比較專案 | 表格式 Q-Learning | DQN |
|:---|:---|:---|
| **狀態表示** | 查詢表 Q[s, a] | 神經網路 Q_θ(s, a) |
| **狀態空間** | 有限離散（如 16 格）| 連續或高維（4D、畫素）|
| **樣本利用** | 每筆只用一次 | Replay Buffer 可多次重用 |
| **訓練穩定性** | 表格更新直接穩定 | 需要 Target Network 穩定 |
| **收斂保證** | 有理論保證（表格情況）| 近似，無嚴格保證 |
| **學習的目標** | Q*（最優，異策略）| Q*（最優，異策略）|

DQN 和 Q-Learning 學的是同一件事（Q*），只是把 Q 表換成神經網路，並加了兩個工程技巧讓訓練可行。

## 11. DQN 的限制

**限制一：過高估計（Overestimation Bias）**  
`max` 操作在有噪聲的估計上會系統性地選到偏高的值。訓練初期 Q 值估計不準，max 會挑到「噪聲最大的那個」，導致 Q 值持續被高估。→ **Double DQN（2015）**解決

**限制二：動作價值 V(s) 和 A(s,a) 耦合**  
Q(s,a) 同時表達「這個狀態好不好」和「這個動作好不好」。在很多狀態下，不管採取什麼動作結果都差不多，此時學習效率低。→ **Dueling DQN（2016）**解決（分離 V 和 Advantage A）

**限制三：均勻回放缺乏優先順序**  
Replay Buffer 隨機取樣——學習進度快的樣本被浪費，TD 誤差大（還沒學好）的重要樣本卻可能長期不被抽到。→ **Prioritized Experience Replay（2015）**解決（PER，Rainbow 的元件之一）

## 12. 結語

DQN 是 2013 年 DeepMind 提出、2015 年在 Nature 發表的里程碑演演算法。核心洞察只有一句話：**Q-Learning 的更新規則完全不用改，只要解決「樣本相關」和「移動標靶」這兩個工程問題，就能把它擴充套件到神經網路。**

從 16 格 FrozenLake 到 Atari 的 2,600 畫素畫面，Q-Learning 的更新公式一字未變。改變的只是：
- Q 表 → 神經網路（讓 Q 能泛化到沒見過的狀態）
- 線上更新 → Replay Buffer（讓樣本獨立）
- 單一網路 → Target Network（讓目標穩定）

這三件事，讓深度強化學習從「概念上可行」變成「真正能用」。

## 13. 方法脈絡 (Lineage)

**建立於**
- Q-Learning (`01_Tabular_Basics/1989_QLearning`) — 繼承完整的異策略 TD 更新規則（Q* = r + γ max Q），只把 Q 表換成神經網路；兩大創新（Replay/Target）是工程解，不改核心演算法

- Dyna-Q (`05_Model-Based/1990_DynaQ`) — 相同的「儲存歷史轉移再取樣更新」精神；Dyna-Q 用模型生成虛擬轉移，DQN 用真實轉移組成 Replay Buffer

| Double DQN | `02_Value_Based_Deep/2015_DoubleDQN` | 解決過高估計：選動作用線上網路，計算 Q 用目標網路，把 max 拆成兩步 |
| Dueling DQN | `02_Value_Based_Deep/2016_DuelingDQN` | 分離 Q = V(s) + A(s,a)，改善不同動作 Q 值差距小時的學習效率 |
| Rainbow DQN | `02_Value_Based_Deep/2017_RainbowDQN` | 整合 DQN + Double + Dueling + PER + N步回報 + NoisyNet + C51，集大成版本 |
| A3C | `02_Value_Based_Deep/2016_A3C` | 用 N 步回報取代單步 TD，非同步多 Worker 取代 Replay Buffer，同樣解決樣本相關問題 |

**橫向對比**

| 演算法 | 資料夾 | 主要差異 |
|:---|:---|:---|
| Q-Learning | `01_Tabular_Basics/1989_QLearning` | 表格（有限離散狀態）vs 神經網路（連續/高維狀態）；DQN = Q-Learning + 工程穩定化 |
| REINFORCE | `03_Policy_Gradient/1992_REINFORCE` | 基於價值（輸出 Q，隱含 greedy 策略）vs 基於策略（直接輸出動作分佈）；DQN 異策略，REINFORCE 同策略 |
