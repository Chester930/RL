# DQN 訓練日誌

環境：CartPole-v1（最大 500 步）
訓練步數：150,000
學習率 α：0.001｜折扣因子 γ：0.99
Replay Buffer：50,000 筆｜Batch size：64
Target 更新頻率：每 500 步｜ε：1.0 → 0.01（前 50,000 步線性衰減）

---

## 一、說明

### 獎勵機制

| 事件 | 獎勵 r |
|:---|:---:|
| 每存活一步 | **+1** |
| 桿子倒下（`\|θ\|` > 0.209 rad ≈ 12°）| 0（直接結束）|
| 小車出界（`\|x\|` > 2.4）| 0（直接結束）|
| 撐過 500 步 | 最後一步仍得 +1，然後結束 |

**稀疏獎勵的難點**：CartPole 沒有「快倒了請注意」的提示獎勵。代理人只能靠「倒下的集數更短」來間接知道自己做錯了，不像 FrozenLake 洞口有明確的 0 獎勵回饋。回報 = 存活步數，所以目標就是**活越久越好**。

---

### ε-greedy 探索（步數對應表）

ε 從 1.0 **線性衰減**到 0.01，前 50,000 步衰減完畢，之後固定在 0.01：

| 步數 | ε 約值 | 行為 |
|:---:|:---:|:---|
| 第 1 ～ 1,000 步 | 1.00 | 完全隨機（只填充緩衝區，不更新網路）|
| 第 10,000 步 | 0.80 | 80% 隨機探索 |
| 第 25,000 步 | 0.51 | 約一半隨機 |
| 第 40,000 步 | 0.21 | 約兩成隨機 |
| 第 50,000 步 | 0.01 | ε 衰減完畢 |
| 第 50,001 步後 | 0.01（固定）| 幾乎完全按策略，僅 1% 隨機 |

> **與 FrozenLake 的差別**：FrozenLake 以「集數」衰減 ε，DQN 以「步數」衰減——因為 DQN 不需要等整集結束才能更新，訓練的計數單位是步數。

---

### DQN 兩大創新

**① 經驗回放（Experience Replay）**

```
每步：  D.push(s, a, r, s', done)           ← 存入回放緩衝區
每步：  batch = D.sample(64)                 ← 隨機抽取迷你批次
        梯度更新 ← 基於 batch（非當前步驟）  ← 打破時間相關性
```

**② 目標網路（Target Network）**

```
每 500 步：Q_theta-  ←  Q_theta    ← 固定目標，避免追移動標靶
計算 TD 目標時：y = r + γ × max_a Q_theta-(s')  ← 用 target_net，非 online_net
```

---

### 損失函式

```
y_i = r_i + γ × max_{a'} Q_theta-(s'_i)    （若 done 則 y_i = r_i）
L   = (1/B) × Σ Huber(y_i − Q_theta(s_i, a_i))

Huber 損失：誤差小時 ≈ MSE（平滑梯度），誤差大時 ≈ L1（有界梯度，防止梯度爆炸）
```

---

### 網路結構（CartPole MLP）

```
輸入（4 維：位置、速度、角度、角速度）
→ 全連線 256 → LayerNorm → ReLU
→ 全連線 256 → LayerNorm → ReLU
→ 全連線 2（輸出：Q(s, ←) 和 Q(s, →)）
```

## 二、緩衝區填充階段（前 1,000 步）

前 1,000 步：ε = 1.0，完全隨機動作，**只收集資料不更新網路**。

| 時刻 | 事件 |
|:---|:---|
| 第 1 步 | 開始收集，緩衝區空（0 筆）|
| 第 1,000 步 | 緩衝區達到 1,000 筆，**首次觸發更新** |

### 第一次網路更新（步數 1,000）

| 指標 | 數值 |
|:---|:---|
| 緩衝區大小 | 1,000 筆 |
| 當前 ε | 1.0000 |
| 初始 loss | 0.500091 |
| 初始 mean Q | -0.0002 |

### 初始 loss = 0.5 的由來

第一次更新時，從緩衝區隨機抽 64 筆，以其中一筆為例：

```
轉移：(s, a, r, s', done=False)

TD 目標（用目標網路計算）：
  max Q_target-(s') ≈ 0.0          ← 網路剛初始化，Q 值接近 0
  y = r + γ × 0.0 = 1.0 + 0 = 1.0

當前 Q 值（線上網路）：
  Q_online(s, a) ≈ 0.0             ← 同樣接近 0

Huber 誤差：
  error = y − Q = 1.0 − 0.0 = 1.0
  Huber(1.0) = 0.5                 ← |error| ≤ 1 時用 MSE 端：error²/2 = 0.5
```

64 筆幾乎每筆都是這樣，平均 loss ≈ **0.5**。
隨著訓練進行，Q 值開始追上真實回報，誤差 y − Q 逐漸縮小。

### θ 角度解讀

```
 步驟 1-5：桿子往左傾（θ ≈ -2.5°）           步驟 6-10：桿子接近垂直（θ ≈ 0°）

         \                                              │
          \  ← θ < 0                                   │  ← θ ≈ 0
           \                                            │
      ┌─────\──┐                                  ┌────┴────┐
      │  小車  │ → 動作：← （推左）               │  小車  │ → 動作：→（推右）
      └────────┘                                  └─────────┘
```

**為什麼桿子往左傾，要往左推？**

推左 → 小車加速向左 → 桿子因慣性「相對往右甩」→ θ 縮小 → 桿子逐漸回正

這和你走路失去平衡時的直覺相同：身體往左傾，腳步往左跨才能追上重心。

**步驟 6-10 的切換**：桿子逐漸回正後（θ 接近 0），代理人改為往右推來止住小車的左移慣性，在垂直附近來回微調——這就是倒立擺控制的核心邏輯。

### 三個里程碑對比

| 步數 | eval 回報 | mean Q | loss | 說明 |
|:---:|:---:|:---:|:---:|:---|
| 50,000 | 28.8 | 95.1 | 5.30 | ε 剛衰減到 0.01，策略不穩定，Q 已被高估 |
| 100,000 | 106.7 | 312.9 | 4.03 | 策略顯著改善，Q 繼續高估（過高估計問題）|
| 150,000 | **500.0** | 224.4 | 3.06 | 滿分！Q 高估仍存在，但策略已達最優 |

**mean Q 為何如此高？**

理論上 γ = 0.99、最多 500 步，完美策略的最大 Q 值約為：

```
Q* ≈ Σ_{t=0}^{499} 0.99^t × 1 = (1 − 0.99^500) / (1 − 0.99) ≈ 99.3
```

但訓練中 mean Q 高達 224 ～ 312，**是理論值的 2–3 倍**。這是 DQN 已知的「過高估計偏差（Overestimation Bias）」：TD 目標中的 `max` 操作在 Q 值有雜訊時會系統性選到偏高的值，造成 Q 被持續高估。這個問題由 **Double DQN（2015）**解決。

**eval 與 recent100 的差距**：

步數 150,000 時，eval = 500 但 recent100 = 122。原因是：
- eval：關閉探索（ε = 0），完全按策略行動 → 滿分
- recent100：訓練時仍有 ε = 0.01 的隨機動作 → 偶爾亂走導致平均拉低

## 五、最終 Q 值快照（訓練完成後）

訓練結束後，對幾個典型狀態查詢 Q 值，驗證策略是否符合直覺：

| 測試狀態 | Q(←) | Q(→) | 選擇動作 |
|:---|:---:|:---:|:---:|
| 正中央靜止（初始） | 247.522 | 247.889 | →（推右） |
| 桿往左傾（需推左） | 246.930 | 246.338 | ←（推左） |
| 桿往右傾（需推右） | 245.962 | 247.204 | →（推右） |
| 小車往左移（需推右） | 223.479 | 222.986 | ←（推左） |

### 逐狀態解讀

**① 正中央靜止** `[x=0, ẋ=0, θ=0, θ̇=0]`

```
Q(←) = 247.522   Q(→) = 247.889   差距 = 0.367
```

桿子完全直立，兩個方向幾乎等價。差距不到 0.4，代表「往哪邊都差不多」——符合直覺。

---

**② 桿往左傾** `[θ = -0.1 rad ≈ -5.7°, θ̇ = -0.1]`

```
Q(←) = 246.930 > Q(→) = 246.338   → 選 ← ✓
```

桿子往左傾且還在加速倒下，推左讓小車追上桿子重心，符合倒立擺控制邏輯。

---

**③ 桿往右傾** `[θ = +0.1 rad ≈ +5.7°, θ̇ = +0.1]`

```
Q(→) = 247.204 > Q(←) = 245.962   → 選 → ✓
```

與②對稱，推右修正右傾，策略完全符合直覺。

---

**④ 小車往左移** `[x = -1.0, ẋ = -0.5, θ = 0, θ̇ = 0]`

```
Q(←) = 223.479 > Q(→) = 222.986   → 選 ← （差距僅 0.49）
```

這個狀態較複雜：桿子完全直立無傾斜，但小車在往左移動。直覺上應推右讓小車折返，但網路選了←，且兩者 Q 值差距僅 0.49（極小）。

兩個觀察：
- **整體 Q 值偏低**（223 vs 247）：網路知道「車在往邊界跑」是危險情況，因此整體回報預期較低。
- **差距極小**：桿子完全垂直時，這步選左或選右對桿子影響相近；接下來幾步的位置修正才是關鍵。這個狀態的不完美是 DQN 已知的限制，Double DQN 仍無法完全解決——位置控制比角度控制更難精確學習。

### DQN 相對於 Q-Learning 的提升

| 問題 | Q-Learning 的困境 | DQN 的解法 |
|:---|:---|:---|
| 連續狀態空間 | 表格存不下 | 神經網路泛化 |
| 樣本時間相關 | 梯度震盪 | Replay Buffer 隨機抽樣 |
| 移動學習目標 | Q 值發散 | Target Network 固定目標 |

### 遺留問題（下一個演算法的起點）

| 問題 | 現象 | 解法 |
|:---|:---|:---|
| 過高估計偏差 | mean Q（224）遠高於理論上限（99）| Double DQN（2015）|
| Q(s,a) 耦合狀態價值與動作優勢 | 相似動作難以分辨優劣 | Dueling DQN（2016）|
| 均勻取樣浪費重要樣本 | 高 TD 誤差的困難轉移很少被抽到 | Prioritized Replay（2015）|
