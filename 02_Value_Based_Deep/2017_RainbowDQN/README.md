# Rainbow DQN | 彩虹 DQN

## 論文

Hessel, M., Modayil, J., van Hasselt, H., Schaul, T., Ostrovski, G., Dabney, W., Horgan, D., Piot, B., Azar, M., & Silver, D. (2017).  
*Rainbow: Combining Improvements in Deep Reinforcement Learning*. AAAI 2018. arXiv:1710.02298.

---

## 六項改進的整合 (Six Improvements Combined)

| 編號 | 改進專案 | 核心思想 | 論文來源 |
|---|------------|---------|-------|
| 1 | **Double DQN** | 解耦動作的選擇與評估 | van Hasselt et al. 2015 |
| 2 | **優先經驗回放 (PER)** | 更頻繁地取樣重要的轉移資料 | Schaul et al. 2015 |
| 3 | **Dueling 網路** | 分離 V(s) 與 A(s,a) 輸出流 | Wang et al. 2016 |
| 4 | **N-步回報 (N-step)** | 使用多步 TD 目標進行學習 | Sutton 1988 |
| 5 | **分散式 RL (C51)** | 對完整的回報分佈進行建模 | Bellemare et al. 2017 |
| 6 | **雜訊網路 (NoisyNet)** | 透過可學習的雜訊權重進行探索 | Fortunato et al. 2017 |

**白話解釋：**

Rainbow 不是一個全新的發明，而是把 2013～2017 年間六個各自獨立的改進全部組合在一起。每一項都解決了原版 DQN 的某個具體缺陷：

```
原版 DQN 的問題                  → 解法

① Q 值被高估（選動作和評估用同一個網路）→ Double DQN：分開用兩個網路
② 每筆資料學習機會相同，浪費了重要資料   → PER：錯得多的資料多抽幾次
③ 每次只更新被選到的動作，其他動作沒學到  → Dueling：V(s) 每次都更新
④ 只看一步後果，學習訊號傳遞慢           → N-步回報：一次看 n 步後果
⑤ 只學「平均回報」，丟失了不確定性資訊   → C51：學整個回報的「分佈」
⑥ ε-greedy 探索太笨，無法針對狀態調整   → NoisyNet：網路內建智慧探索
```

這六項中，① ② ③ 我們已經在前面的章節分別學過了。④ 在 N-Step TD 也見過。⑤ 和 ⑥ 是 Rainbow 帶來的新概念。

## 分散式強化學習 (Distributional RL, C51)

不同於傳統方法僅學習期望回報 $E[G]$，C51 使用 51 個離散的「原子」(Atoms) 來學習完整的回報分佈 $P(G)$：

```
支撐集 (Support): z_i = v_min + i * (v_max - v_min) / (N-1)    針對 i = 0..N-1

網路輸出: p(s, a) ∈ Delta^N    (原子上的機率分佈，經由 softmax)

Q(s, a) = sum_i z_i * p_i(s, a)     (從分佈中計算期望值)
```

分散式貝爾曼運算元 (Distributional Bellman operator)：
```
T z_i = r + gamma * z_i    (將位移後的原子投影回支撐集)
```

**白話解釋：**

**傳統 DQN 學的是什麼？**

```
Q(s, ←) = 85   ← 「從這個狀態選往左，預期能活 85 步」
```

這是一個純量（單一數字）。它告訴你「平均」能活多久，但不告訴你這個 85 是「穩穩的 85」還是「有時候 10、有時候 160，平均剛好 85」。

**C51 學的是什麼？**

C51 不學那個 85，而是學一個「長條圖」：

```
C51 輸出（51 個原子的機率）：

往左：
  0步機率  = 5%
  10步機率 = 3%
  ...
  80步機率 = 12%  ← 峰值在這
  90步機率 = 10%
  ...
  500步機率 = 2%

Q(s, ←) = 0×5% + 10×3% + ... + 80×12% + ... = 85  ← 取期望值才得到 85
```

**為什麼長條圖比單一數字有用？**

```
動作A：穩定，每次都活 80～90 步  （分佈集中）
動作B：不穩定，50% 活 10 步，50% 活 160 步  （分佈雙峰）
兩者期望值都是 85，但風險完全不同！
```

傳統 DQN 看到「A=85，B=85」，無法區分。C51 看到「A 集中，B 分散」，可以選擇比較穩定的動作。

**51 個原子是什麼？**

把回報範圍（本實作 0 到 500）均勻切成 51 格：

```
原子0  = 0     （代表「活 0 步」這個回報值）
原子1  = 10    （代表「活 10 步」）
原子2  = 20
...
原子50 = 500   （代表「活 500 步，滿分」）

網路輸出每個原子的機率：p0, p1, ..., p50
（51個數字，加總 = 1，就是機率分佈）
```

**貝爾曼投影是什麼？**

傳統 DQN 的學習目標：`Q ← r + γ × Q_target`（把目標值往前移一步）

C51 的做法相同，只是目標是整個分佈而非單一數字：

```
原子 z_i 的新位置 = r + γ × z_i
                  （把每個原子往前推一步）

但推完之後原子可能不在原本的格子上
→ 需要把機率「投影」回最近的兩個格子分配
→ 這個步驟叫「分佈貝爾曼投影」
```

類比：你有一把尺，刻度是 0, 10, 20...。你算出答案是 85，但尺上沒有 85 這格——你把這個答案的重量分配到 80（50%）和 90（50%）這兩個刻度上。

## 雜訊網路 (NoisyNet)

用可學習的隨機權重取代傳統的 Epsilon-greedy 策略：
```
y = (mu_w + sigma_w * epsilon) x + (mu_b + sigma_b * epsilon)
```

探索行為會從權重的不確定性中自然產生，並能根據任務自動調整探索強度。

**白話解釋：**

**ε-greedy 的問題是什麼？**

傳統 DQN 的探索方式：
```
以機率 ε 隨機選動作（探索）
以機率 1-ε 選最好的動作（利用）

ε 從 1.0 線性衰減到 0.01
```

問題：
```
① ε 是全域性設定，不管在什麼狀態都用同樣的探索力度
   → 在「已經學得很好」的狀態仍然 1% 機率亂選
   → 在「完全陌生」的狀態也只有固定機率探索

② ε 衰減排程需要手動設計（何時衰減？衰減多快？）
```

**NoisyNet 怎麼做？**

普通線性層的公式：
```
y = W × x + b    （W 和 b 是固定的學習引數）
```

NoisyLinear 的公式：
```
y = (μ_W + σ_W ⊙ ε_W) × x + (μ_b + σ_b ⊙ ε_b)
     ↑ 均值      ↑ 雜訊強度  ↑ 隨機雜訊
```

- `μ`（mu）：正常的學習引數，就是普通網路的 W 和 b
- `σ`（sigma）：額外學習的「雜訊強度」，控制這層要加多少隨機性
- `ε`（epsilon）：每次前向傳遞重新取樣的隨機數（不是被學習的）

**關鍵：`σ` 是被訓練的**

```
訓練初期：σ 大 → 網路輸出很隨機 → 大量探索
訓練後期：網路學到 σ 應該小 → 輸出趨向確定 → 少量探索

探索力度根據任務難度自動調整，不需要手動設計排程
```

**不同狀態，不同探索強度：**

```
狀態A（熟悉的安全狀態）：
  網路在這種狀態見過很多次，σ 自動學到很小
  → 這個狀態不需要探索，直接選最好的動作

狀態B（陌生的危險狀態）：
  網路在這種狀態見過很少，σ 仍然較大
  → 這個狀態需要探索，保持隨機性
```

類比：老手開車遇到熟悉的路口（σ 小，直接開）；遇到陌生的山路彎道（σ 大，謹慎摸索）。ε-greedy 的老手和新手在每個路口都有同樣機率隨機轉彎——不合理。

## 消融實驗結果 (Ablation Results)

Rainbow 論文在 Atari-57 基準測試上對各元件進行了消融實驗：

```
完整 Rainbow > -PER > -Dueling > -NoisyNet > -N-step > -Distributional > -Double
```

其中 **N-步回報** 與 **分散式 RL** 的貢獻最為顯著；**Double DQN** 在整合了其他技術後，單獨的貢獻相對較小。

**白話解釋：**

「消融實驗」的意思是：每次拿掉一個元件，看整體效果掉多少。掉越多代表那個元件越重要。

```
結果（從最重要到最不重要）：

移除 Distributional（C51）→ 效果掉最多  ← 最重要！
移除 N-step            → 效果掉很多    ← 第二重要
移除 NoisyNet          → 效果掉不少
移除 Dueling           → 效果掉一些
移除 PER               → 效果掉一點
移除 Double DQN        → 效果掉最少    ← 在 Rainbow 裡最不重要
```

有趣的觀察：Double DQN 單獨使用時效果很好，但在 Rainbow 裡貢獻最小。這是因為其他元件（特別是 C51 的分散式學習）已經間接解決了 Q 值高估的問題。

## 關鍵超引數 (Key Hyperparameters)

| 引數 | 數值 | 備註 |
|-----------|-------|--------|
| 原子數量 (N atoms) | 51 | |
| V_min / V_max | -10 / 10 | 分佈範圍（Atari 設定；CartPole 用 0/500）|
| N step | 3 | 多步回報步數 |
| PER alpha | 0.5 | |
| PER beta (初始) | 0.4 | |
| Noisy sigma_0 | 0.5 | 初始雜訊強度 |
| 目標更新頻率 | 8000 步 | |

**白話解釋：**

**原子數量 51**：把回報範圍切成 51 格。為什麼是 51？原論文嘗試了幾個數字，51 效果最好，且奇數讓中間有一個剛好在 0 的原子（對稱）。

**V_min / V_max**：長條圖的左右邊界。設定太窄（如 0～10）就裝不下大的回報；太寬（如 -1000～1000）每個格子太稀疏，表達能力差。CartPole 回報是 0～500，就設 [0, 500]。

**N step = 3**：每筆資料用實際的 r_t、r_{t+1}、r_{t+2}，然後才用網路估計剩下的。n=1 太短（全靠估計），n=∞ 太長（全靠實際但方差大），n=3 是論文找到的甜蜜點。

**sigma_0 = 0.5**：NoisyNet 所有層的雜訊強度初始值。0.5 代表「一開始雜訊和訊號一樣強」——保證訓練初期有足夠探索。隨訓練進行，σ 會自動學到合適的值。

## 六項改進的互動關係

```
             ┌─────────────────────────────────────────┐
             │           Rainbow DQN                   │
             │                                         │
  「學什麼」  │  C51 分散式：學回報分佈而非純量         │
             │  Double DQN：正確計算目標分佈            │
             │  Dueling：V(s) 和 A(s,a) 分流學習       │
             │                                         │
  「學哪些」  │  PER：優先學習錯誤多的資料              │
             │  N-step：每筆資料涵蓋 n 步的資訊         │
             │                                         │
  「怎麼探索」│  NoisyNet：智慧探索，取代 ε-greedy      │
             └─────────────────────────────────────────┘
```

這六項不是隨便疊加的——它們分別負責「學習目標的品質」、「資料的選擇」、「探索的方式」，三個面向各司其職，組合起來才能發揮最大效果。
