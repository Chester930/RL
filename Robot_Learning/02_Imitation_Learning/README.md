# 02 — 模仿學習（2004–2019）

> **時代定位**：解決「獎勵函數難設計」問題，直接從人類示範學習  
> **核心突破**：讓機器人無需明確獎勵信號也能習得複雜行為

---

## 為什麼需要模仿學習

```
傳統 RL 的困境：
  問題：「抓住雞蛋但不打破它」的獎勵函數怎麼寫？
        力道 < 3N 且 接觸點準確 且 不滑落 且...
  → 獎勵函數工程量巨大，且容易出現 reward hacking

模仿學習的出發點：
  既然人能做這個動作，直接學人怎麼做不就好了？
  → 從示範資料中學習行為，不需要明確定義「什麼是好」
```

---

## 方法譜系

```
模仿學習家族
├── BC（行為複製）    ← 最簡單，直接監督學習
│     問題：分佈偏移
├── DAgger (2011)     ← BC 改良：互動式示範收集
├── GAIL (2016)       ← GAN 框架，學示範的分佈
└── IRL（反 RL）      ← 推導獎勵函數
      ├── MaxEnt IRL (2006) ← 最大熵原則
      └── AIRL (2018)       ← 解耦獎勵與動態
```

---

## 本章節演算法

### 2004_BC — Behavioral Cloning
```
原理：把模仿學習當成監督學習
  輸入 sₜ → 輸出 aₜ = π_θ(sₜ)
  損失：L = ||π_θ(s) - a_expert||²

本專案實作：BC on Pendulum-v1
  示範來源：訓練好的 SAC 策略（5000 步示範）
  eval：-144.6（明顯差於 SAC 的 -171.8）
  原因：分佈偏移（訓練示範 ≠ 測試遇到的狀態）

核心限制：
  訓練：s ∈ 示範分佈 → 學好
  測試：s 略微偏移 → 誤差 → 更大偏移 → 崩潰（誤差複合）
```

### 2016_GAIL — Generative Adversarial Imitation Learning
```
論文：Ho & Ermon (2016), arXiv:1606.03476

核心洞察：模仿學習 ≡ 讓策略的 state-action 分佈 = 示範的分佈
  → 用 GAN 框架解決！

架構：
  生成器（Generator） = 策略 π
    任務：生成 (s,a) 讓判別器無法辨別
  判別器（Discriminator） = D(s,a)
    任務：區分「示範的 (s,a)」和「策略的 (s,a)」

訓練迴圈：
  for each iteration:
    1. 用策略 π 收集軌跡
    2. 更新 D：區分示範 vs 策略
    3. 用 -log D(s,a) 作為 reward
    4. 更新 π（PPO/TRPO）使 reward 最大
```

### 2006_MaxEntIRL — Maximum Entropy IRL
```
論文：Ziebart et al. (2008), AAAI

問題：IRL 的歧義性（多個獎勵函數都能解釋同一個示範）
解法：最大熵原則 → 在所有可能的獎勵函數中，選擇使行為分佈熵最大的那個

優點：
  - 得到的獎勵函數可泛化到新場景
  - 不過擬合示範的特定路徑（允許等效路徑）

機器人應用：
  - 學習人的移動意圖（預測未來軌跡）
  - 無人車行為複製
  - 靈巧手操作策略提取
```

---

## BC vs GAIL vs IRL 對比

| 方法 | 示範需求 | 泛化能力 | 計算成本 | 適用 |
|---|---|---|---|---|
| BC | 多（>1000） | 低（分佈偏移）| 極低 | 快速基線、簡單任務 |
| GAIL | 中（>100） | 中 | 中（需線上互動）| 複雜連續任務 |
| MaxEnt IRL | 少（10-50）| 高 | 高 | 需要可泛化獎勵函數 |

## 模仿學習在現代 VLA 中的角色

```
模仿學習並未過時！PI0、RT-2 的訓練都大量依賴：
  人工示範資料（遙控操作、VR 手套示範）
  + 大規模 BC 預訓練
  + 少量 RL 精修

現代版 BC = 大規模示範 + Diffusion Policy / Flow Matching
               ↑ 解決多模態問題（多種等效抓法）
```
