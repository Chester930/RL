# 機器人學習技術演進全景

---

## 四個時代

### Era 1：傳統強化學習（2013–2018）

**核心問題**：如何讓機器人從環境互動中學習連續動作控制？

```
挑戰：
  - 高維連續動作空間（關節角度、末端執行器位置）
  - 稀疏或延遲的獎勵信號
  - 樣本效率極低（需數百萬次互動）
  - 難以在真實機器人上部署（昂貴、危險）

突破：
  DDPG (2015)：確定性策略梯度 + Replay Buffer → 連續動作可學
  TD3  (2018)：雙 Critic + 延遲更新 → 訓練穩定性大幅提升
  SAC  (2018)：最大熵 RL → 樣本效率最高，不易過擬合
  HER  (2017)：事後目標重標記 → 稀疏獎勵操作任務可學
```

**代表成果**：機器手臂能學會 Reach/Push/Pick（Gym Robotics 環境）

---

### Era 2：模仿 + 目標學習（2016–2022）

**核心問題**：如何減少對精心設計獎勵函數的依賴？

```
挑戰：
  - 獎勵函數設計困難（精確抓取的「成功」如何定義？）
  - 純 RL 探索成本高（真實機器人磨損、危險）
  - 長序列任務中 Credit Assignment 困難

突破：
  BC (2004)：直接複製示範 → 最簡單的切入點
  GAIL (2016)：GAN 框架學習示範分佈 → 解決 BC 的分佈偏移
  IRL  (MaxEnt)：反推獎勵函數 → 可泛化的行為複製
  HER+ Goal-RL (2017+)：任意目標泛化 → 一個模型學多種任務
```

**代表成果**：OpenAI 機械手（Dactyl）學會魔方旋轉

---

### Era 3：大模型進入機器人（2022–2024）

**核心問題**：如何讓機器人理解自然語言指令並泛化到新任務？

```
挑戰：
  - 以往方法泛化能力差（換一個物體就失敗）
  - 語言理解和動作執行脫節
  - 每個新任務都需要重新訓練
  - 多模態感知（視覺 + 語言 + 觸覺）整合困難

突破：
  RT-1  (2022)：大規模機器人資料 + Transformer → 泛化能力初現
  RT-2  (2023)：VLM 預訓練 + 機器人微調 → 語言理解飛躍
  Diffusion Policy (2023)：擴散模型生成動作 → 多模態操作
  OpenVLA (2024)：開源 VLA 基準 → 社群可複現

機器人資料規模飛躍：
  DDPG 時代：數千次環境互動
  RT-1 時代：130,000 次真實機器人示範
  π₀ 時代：10,000+ 小時、7 種機器人、68 種任務
```

**代表成果**：機器人能執行「把藍色積木放進紅色容器」這類語言指令

---

### Era 4：基礎模型 + RL 精修（2024–2026）

**核心問題**：通用基礎模型 + 精確操作如何結合？

```
挑戰：
  - VLA 大模型通用但不夠精確（「最後一毫米」問題）
  - 重訓整個大模型成本極高
  - 高精度任務（螺絲、精密插拔）需要微小誤差
  - 真實世界部署需要快速適應

突破：
  π₀ (2024)：Flow Matching + Action Expert → 通用操作基礎
  π₀-FAST (2024)：動作 Token 化 → 推理加速
  RLT (2026)：RL Token + 小型 Actor-Critic → 精修不改大模型
```

**代表成果**：機器人 15 分鐘學會鎖 3mm M3 螺絲，速度超越人工遙控

## 關鍵洞察

> **問題從「如何學」轉移到「如何有效利用預訓練知識」**

1. **2015** — 問題：機器人能用 RL 學習嗎？→ 答：能，但很慢
2. **2018** — 問題：如何讓稀疏任務可學？→ 答：HER 事後目標
3. **2022** — 問題：如何跨任務泛化？→ 答：用 LLM/VLM 做預訓練
4. **2024** — 問題：通用性和精確性能兼顧嗎？→ 答：PI0 的分工架構
5. **2026** — 問題：如何讓大模型快速適應高精度？→ 答：凍結大模型，RL Token 精修
