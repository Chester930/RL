# 08 — RL 精修 VLA（2026）

> **時代定位**：用輕量 RL 解決 VLA 大模型的「最後一毫米」問題  
> **核心突破**：凍結大模型 + RL Token 作為介面 → 分鐘級高精度學習

---

## 問題定義：最後一毫米

```
VLA（π₀ 等）的強項：
  ✅ 理解語言指令
  ✅ 視覺感知和場景理解
  ✅ 通用操作（拿、放、推）
  ✅ 多任務泛化

VLA 的弱點：
  ❌ 精確操作（誤差 > 1mm 就失敗的任務）
     例：M3 螺絲鎖入（螺紋對準精度 ±0.5mm）
         RJ45 網路線插入（卡扣對準）
         USB-C 插入（方向 + 位置雙重精度）

「最後一毫米」問題：
  大模型輸出「差不多對的動作」，但不夠精確
  重訓整個大模型成本極高（數週 GPU 時間）
```

---

## 本章節演算法

### 2026_RLT — RL Token

```
論文：Cetin, Zhao, Tang (2026), arXiv:2604.23073
機構：Physical Intelligence
官網：pi.website/research/rlt

核心設計哲學：
  「大模型負責理解，小網路負責精修」
  「不動大模型，只在輸出介面加 RL」

完整架構：
┌────────────────────────────────────────────────────┐
│               π₀ VLA（完全凍結）                   │
│                                                     │
│  RGB × 3 + 語言指令 + 本體感知                     │
│            ↓ （前向計算）                           │
│      ┌─────────────────┐                            │
│      │   RL TOKEN      │ ← 新增：從 VLA 中萃取      │
│      │  緊湊特徵向量   │   任務關鍵資訊              │
│      └────────┬────────┘                            │
└───────────────┼────────────────────────────────────┘
                │ （VLA 凍結，只輸出 RL Token）
    ┌───────────▼────────────────┐
    │     Actor-Critic 頭        │
    │                            │
    │  Actor：RL Token → Δaction │
    │  Critic：RL Token → V(s)   │
    │                            │
    │  特性：                    │
    │  - 參數極少（<<VLA）       │
    │  - 離線策略（Replay Buffer）│
    │  - 每秒數百次更新           │
    └───────────┬────────────────┘
                │
         精修後動作 = VLA 基礎動作 + Δaction
```

### RL Token 的技術細節

```
什麼是 RL Token？
  從 VLA 的中間層萃取的一個向量表示
  設計目標：壓縮「當前觀測中與 RL 決策相關的資訊」
  實作：在 VLA 架構中添加額外的讀出（readout）頭
         輸出固定維度向量 e ∈ ℝᵈ

為什麼不直接用 VLA 的動作輸出？
  VLA 動作輸出 = 50 步 Chunk，不適合逐步 RL 更新
  RL Token = 壓縮的當前狀態表示，適合 Actor-Critic 使用
```

### 兩階段訓練

```
Stage 1：SFT 適應（讓 VLA 學會輸出 RL Token）
  資料：少量任務示範（數百次）
  目標：
    a. VLA 能輸出有意義的 RL Token
    b. 基礎策略已「差不多對」（以便 RL 精修）
  時間：數小時

Stage 2：線上 RL 精修（VLA 完全凍結）
  環境：真實機器人 or 高保真模擬器
  演算法：離線策略 Actor-Critic（SAC 變體）
  Replay Buffer：儲存過去經驗
  更新頻率：每秒數百次（小網路，速度快）
  收斂時間：15 分鐘 ~ 數小時
```

### 獎勵設計

```
任務完成信號（稀疏 or 密集）：
  螺絲鎖入：扭矩感應器達到設定值 → +1
  插入任務：電氣接觸成功 → +1

可選的輔助獎勵：
  位置誤差懲罰：||end_effector - target||²
  速度平滑懲罰：||v_t - v_{t-1}||²
```

## RLT 的深遠意義

```
技術意義：
  證明「大模型凍結 + 小型 RL 介面」可行
  開創了 VLA 精修的標準正規範式

工程意義：
  無需 GPU 叢集重訓 → 可在邊緣部署
  快速適應新任務 → 工廠換線不需要等待幾天的重訓

哲學意義：
  大模型提供「知識」（理解世界）
  小型 RL 提供「技能」（精確執行）
  兩者分工明確，各司其職
```

## 與傳統方法的根本對比

```
傳統線上 RL（from scratch）：
  隨機初始化 → 隨機探索 → 稀疏獎勵 → 數百萬步才收斂

RLT：
  π₀ 已知「大概怎麼操作」（來自 10,000 小時示範）
       ↓ RL Token 萃取關鍵狀態資訊
       ↓ Actor-Critic 只需學「最後的精修量」
  結果：數百步收斂（15 分鐘！）

類比：
  傳統 RL = 從零開始教嬰兒學走路（年）
  RLT = 教已經會走路的人學鋼索平衡（天）
```
