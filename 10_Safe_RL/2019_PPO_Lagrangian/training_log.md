# PPO-Lagrangian 訓練日誌

> 環境：SafePendulum（Pendulum-v1 + 代價信號）  
> 安全約束：|θ̇| > 2.0 rad/s → cost=1，每集限 **5 步**違規  
> 訓練規模：300K 步，n_steps=2048（146 次更新）  
> 裝置：CPU（PyTorch 2.11.0+cpu）

---

## 演算法特有指標

| 指標 | 說明 |
|---|---|
| `eval/mean_reward` | 評估時每集平均獎勵（越高越好，目標 ≥ -300） |
| `eval/mean_cost` | 評估時每集平均違規步數（越低越好，目標 ≤ 5） |
| `eval/lambda` | 評估時拉格朗日乘數 λ（反映安全約束緊張程度） |
| `train/lambda` | 訓練時 λ 的即時值 |
| `train/ep_cost` | 訓練中每集的違規步數 |

## 訓練結果（eval）

| 步數 | eval 獎勵 | eval 代價/集 | λ 值 | 安全？ |
|---|---|---|---|---|
| 0（繼承舊策略） | -1378.5 | ~7.4 | 0.0000 | ✓（限 25）→ ✗（限 5） |
| 206K（cost_limit=5 生效後 +1st） | -1448.6 | 9.7 | **3.1127** | ✗ |
| 300K（最終） | -1817.7 | 8.9 | **5.6818** | ✗ |

**重要**：前 100 次更新使用 cost_limit=25（λ=0 從未啟動）。  
改為 cost_limit=5 後，λ 立即從 0 跳至 3.11，展示 Lagrangian 機制的啟動動態。

---

## Lagrangian 乘數 λ 動態

```
cost_limit=25（前 100 次更新）：
  λ = 0.0000  → 代價 7.4 ≤ 25，約束不活躍，λ 不更新

cost_limit=5（第 101–146 次更新）：
  update 120：代價 9.7 > 5  → λ 更新：0 → 3.1127（上升！）
  update 140：代價 8.9 > 5  → λ 更新：3.11 → 5.6818（持續上升！）
```

**解讀**：λ 的上升代表演算法正在對安全違規施加越來越高的懲罰。若繼續訓練，λ 增大會強迫策略降低代價，代價以損失獎勵為代價。

---

## 安全 vs 獎勵 Trade-off

```
λ 啟動前（舊策略，cost_limit=25）：
  獎勵：-1378.5  →  代價：7.4  →  λ：0.000（沒有安全壓力）

λ 啟動後（cost_limit=5）：
  更新 1：獎勵 -1448.6  代價 9.7  λ 3.11
  更新 2：獎勵 -1817.7  代價 8.9  λ 5.68（代價稍降，但獎勵下滑更多）
```

獎勵從 -1378 → -1817（下降 32%）而代價只從 9.7 → 8.9（下降 8%），反映 Lagrangian 方法在收斂前有明顯的安全-獎勵折衷陣痛期。

---

## 觀察與分析

**λ 動態**：cost_limit 從 25 改為 5 後，λ 從 0 立刻跳升，完美展示 Lagrangian 懲罰機制的啟動。lr_lambda=0.05 使 λ 更新相對激進（可視需要降低）。

**獎勵-安全 trade-off**：λ 上升導致有效損失 = reward_loss + λ × cost_loss，策略被迫在獎勵和安全間取捨。短期內獎勵明顯惡化（-1379→-1818），是過渡期現象。

**與 PPO 基線的差異**：PPO（無安全約束）在 Pendulum 上通常可達 -200 以內。加入 Lagrangian 後，學習到的策略需要犧牲部分獎勵來滿足 |θ̇| ≤ 2.0 的約束，這是安全強化學習的核心代價。

**CPU 限制**：300K 步耗時約 6 分鐘。要完整展示 λ 收斂到穩態、代價降至限制以下，預計需要 1M 步以上（GPU 環境）。

## 對比：PPO-Lagrangian vs CPO

| 項目 | PPO-Lagrangian | CPO |
|---|---|---|
| 更新機制 | 梯度剪裁 + λ 軟約束 | 自然梯度 + 硬約束（信任域） |
| eval 代價最低 | 8.9/集 | **6.5**/集 |
| 最終 eval 獎勵 | -1817.7 | -1483.7 |
| λ / 約束強制 | λ 從 0→5.68（動態） | 無 λ（硬約束） |
| 安全保證強度 | 軟（λ 需時間收斂） | 較強（每步信任域限制） |
| 訓練速度（300K）| ~6 min CPU | ~6 min CPU（但每次 update 更慢） |
| 教學亮點 | λ 啟動動態清晰可見 | 全程 CPO 模式展示嚴苛約束效果 |
