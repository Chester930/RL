## 說明

### CartPole-v1 環境

```
觀測值（連續 4 維）：
  x         ：小車位置（-4.8 ~ 4.8）
  ẋ         ：小車速度
  θ         ：桿子角度（-24° ~ 24°）
  θ̇         ：桿子角速度

動作（離散 2 維）：
  0 = 向左推力
  1 = 向右推力

獎勵：每活過一步得 r = 1.0
終止條件：桿子傾斜超過 ±12° 或小車超出邊界，或達到 500 步
```

里程碑標準：單集回報 ≥ 100 步（CartPole 官方解決標準為平均 195，REINFORCE 高方差，先記 100 步為首個里程碑）

### REINFORCE 更新流程

```
① 完整集數收集（必須走完才能更新）
   for t = 0, 1, ..., T-1:
       a_t ~ π(·|s_t; θ)          # 從策略取樣動作
       記錄 log π(a_t|s_t; θ)

② 倒序計算折扣回報（蒙特卡羅）
   G_T = 0
   for t = T-1, T-2, ..., 0:
       G_t = r_{t+1} + γ × G_{t+1}

③ 回報歸一化（方差縮減）
   G_t ← (G_t - mean(G)) / (std(G) + ε)

④ 減去基準值（可選，方差縮減）
   advantage_t = G_t - mean(G)

⑤ 策略梯度損失 + 更新
   L = -Σ_t advantage_t × log π(a_t|s_t; θ)
   θ ← θ - α × ∇L
```

### 網路結構

```
輸入（4 維）→ 全連線 128 → ReLU → 全連線 128 → ReLU → 輸出（2 維 logits）
→ Categorical 分佈 → 取樣動作
```

### REINFORCE 的核心限制

1. **高方差**：G_t 是整集所有隨機事件的累積，同一狀態不同集的 G_t 差異很大。
   歸一化和基準值只是部分緩解——Actor-Critic 用 V(s) 作為基準，方差更低。

2. **樣本效率低**：每條軌跡收集後只用一次梯度更新，然後丟棄。
   PPO 的 clip ratio 允許每條軌跡做多次更新。

3. **必須等整集結束**：無法做線上（on-step）學習。
   TD 系列方法（SARSA、A2C）每步更新，不需要等整集。

## REINFORCE 核心機制詳解

> 以下使用訓練的實際參數（γ=0.99、lr=0.001、use_baseline=True）與訓練資料說明 REINFORCE 失敗的根本原因

---

### 一、策略梯度公式

```
REINFORCE 更新公式：
  θ ← θ + α × G_t × ∇_θ log π(a_t | s_t; θ)

直覺：
  G_t > 0（好的結果）→ 提高此動作的對數機率
  G_t < 0（差的結果）→ 降低此動作的對數機率
  G_t 的大小決定「梯度步幅」

使用 baseline（mean(G)）版本：
  advantage_t = G_t - mean(G)
  θ ← θ + α × advantage_t × ∇_θ log π(a_t | s_t; θ)
```

---

### 二、高方差問題：具體數字說明

**同一狀態（桿子 θ ≈ -3°）的 G_t 在不同集數差異**

CartPole 在訓練初期（ep 4762 前），策略不穩定，集數長度差異很大：

| 集數 | 集長（步數）| G_t（t=5，5 步後的折扣回報）| 基準值 mean(G) | advantage_t |
|---|---|---|---|---|
| 短集（17 步，早期典型）| 17 | ~11.7 | ~9.0 | **+2.7** |
| 長集（108 步，ep 4762）| 108 | ~64.5 | ~39.3 | **+25.2** |
| 差異倍數 | — | **5.5×** | — | **9.3×** |

**梯度受 G_t 縮放的影響：**

```
短集（17步）：advantage ≈ +2.7  → 梯度步幅 = 0.001 × 2.7 = 0.0027
長集（108步）：advantage ≈ +25.2 → 梯度步幅 = 0.001 × 25.2 = 0.0252

→ 即使是同一個「好動作」，因為所在集的長度不同，梯度相差 9×
→ 策略學習方向正確，但步幅忽大忽小 → 訓練不穩定
```

**為什麼 CartPole 初期集數很短（≈10步）：**

```
策略隨機初始化（logits ≈ 0）→ 動作接近 0.5/0.5 均勻
→ 桿子靠隨機搖晃維持直立，平均 ~10 步就倒
→ G_0 ≈ 10，advantage ≈ 0（所有集都差不多短）
→ 梯度幾乎為 0，學習速度極慢

對比第 4762 集（108步）：梯度終於有意義，但之後又回到短集 → 無法持續
```

### 三、與 PPO / A2C 的對比（CartPole 5000 集）

| 面向 | REINFORCE | A2C | PPO |
|---|---|---|---|
| 基準值 | mean(G)（全集均值）| V(s)（逐步估計）| V(s)（逐步估計）|
| 優勢估計 | G_t − mean(G)（高方差）| r + γV(s') − V(s)（TD 誤差，低方差）| GAE(γ=0.99, λ=0.95）|
| 每集資料重用 | 1 次 | 1 次（或 n-step）| **10 次（n_epochs）** |
| CartPole 5K 步 eval | **9.5±0.5（失敗）** | ~150（不穩定）| ~500（滿分）|

**根本原因對比：**

```
REINFORCE 的 G_t：
  G_t = r_{t+1} + γr_{t+2} + γ²r_{t+3} + ...  （整集累加，高方差）

A2C 的 advantage（TD 誤差）：
  δ_t = r_{t+1} + γV(s_{t+1}) − V(s_t)        （僅一步估計，低方差）

PPO 的 GAE：
  A_t = δ_t + γλδ_{t+1} + (γλ)²δ_{t+2} + ...  （折衷：λ=0.95 偏向蒙特卡羅）

方差大小：G_t > GAE > δ_t
偏差大小：G_t < GAE < δ_t（越短的 bootstrap 偏差越大）
PPO 的 GAE 在偏差-方差之間取得最佳折衷
```

### 四、REINFORCE 失敗的直接診斷

```
觀察到的現象（訓練日誌）：
  ep 500–4000：eval 穩定在 9.0–9.7（幾乎不動）
  ep 4500：eval 暫時升至 27.3
  ep 5000：eval 又回到 9.5

診斷：
  1. 初期集數太短（~10步）→ G_t ≈ 10 → advantage ≈ 0 → 梯度≈0
  2. 梯度幾乎為 0 → 策略幾乎不更新 → 集數維持在 ~10 步
  3. 偶爾出現長集（ep 4762, 108步）→ 較大梯度 → 短暫改善
  4. 但新策略不穩定 → 下一集又短 → 梯度又消失

這是「低回報-低梯度-低回報」的惡性循環，典型的 REINFORCE 失敗模式。
PPO 用 clip ratio 的重複使用（10 epoch）打破了這個循環。
```
