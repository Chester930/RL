## PPO 核心機制詳解

> 以下計算使用 CartPole 訓練的實際參數（clip_eps=0.2、gamma=0.99、gae_lambda=0.95、n_steps=2048、n_epochs=10）

---

### 一、GAE 優勢估計計算過程

GAE（Generalized Advantage Estimation）解決了蒙特卡羅（高方差）與 TD(0)（高偏差）的折衷問題。

**公式：**

```
delta_t   = r_t + γ · V(s_{t+1}) - V(s_t)       ← 一步 TD 誤差
A_t^{GAE} = delta_t + (γλ) · delta_{t+1}
                    + (γλ)² · delta_{t+2} + ...
           = Σ_{k=0}^{T-t} (γλ)^k · delta_{t+k}

γ = 0.99, λ = 0.95  →  γλ = 0.9405
```

**CartPole 5 步具體計算範例（初始訓練期，V(s) 約為 5-10）：**

假設某段 5 步軌跡：

| 步驟 t | s_t（角度°）| a_t | r_t | V(s_t) | V(s_{t+1}) | delta_t |
|---|---|---|---|---|---|---|
| 0 | -1.5° | ← | 1.0 | 6.2 | 7.1 | 1.0 + 0.99×7.1 − 6.2 = **+1.83** |
| 1 | -0.8° | → | 1.0 | 7.1 | 8.5 | 1.0 + 0.99×8.5 − 7.1 = **+2.32** |
| 2 | -0.2° | → | 1.0 | 8.5 | 9.3 | 1.0 + 0.99×9.3 − 8.5 = **+1.71** |
| 3 | +0.4° | ← | 1.0 | 9.3 | 8.8 | 1.0 + 0.99×8.8 − 9.3 = **+0.41** |
| 4 | +0.1° | ← | 1.0 | 8.8 | — | 1.0 − 8.8 = **-7.80**（集數結束）|

從 t=4 倒序計算：

```
A_4 = delta_4 = -7.80

A_3 = delta_3 + γλ · A_4
    = 0.41 + 0.9405 × (-7.80)
    = 0.41 − 7.34 = -6.93

A_2 = delta_2 + γλ · A_3
    = 1.71 + 0.9405 × (-6.93)
    = 1.71 − 6.52 = -4.81

A_1 = delta_1 + γλ · A_2
    = 2.32 + 0.9405 × (-4.81)
    = 2.32 − 4.52 = -2.20

A_0 = delta_0 + γλ · A_1
    = 1.83 + 0.9405 × (-2.20)
    = 1.83 − 2.07 = -0.24
```

**解讀：**
- t=0、1、2 的 delta 都是正的（局部有利），但 GAE 「看到了後面集數結束」，A_0 仍為負。
- 這正是 GAE 的優勢：比純 TD(0)（只看 delta_0=1.83）更準確地評估「此時的動作是否真的好」。
- λ 接近 1.0 時 GAE → 蒙特卡羅回報；λ=0 時 GAE → TD(0)。λ=0.95 是常用折衷。

**歸一化（Normalize Advantages）：**

```python
b_advantages = (b_advantages - b_advantages.mean()) / (b_advantages.std() + 1e-8)
```

歸一化讓 2048 步的優勢分佈在 0 附近，防止梯度量級隨回報規模變化。

### 二、Clip 操作實際計算範例

**核心公式：**

```
ratio_t = π_new(a_t | s_t) / π_old(a_t | s_t)
        = exp(log π_new − log π_old)

L_CLIP = min( ratio · A, clip(ratio, 1−ε, 1+ε) · A )
       = min( surr1, surr2 )

ε = 0.2  →  clip 範圍 [0.8, 1.2]
```

**情況 A：好動作（A > 0）、ratio 過高（更新過大）**

```
情境：某步推左讓桿子回正，A = +4.2
      第 1 個 epoch 後：log π_new = −0.3，log π_old = −1.0
      ratio = exp(−0.3 − (−1.0)) = exp(0.7) = 2.01

surr1 = 2.01 × 4.2 = 8.44
surr2 = clip(2.01, 0.8, 1.2) × 4.2 = 1.2 × 4.2 = 5.04

L_CLIP = min(8.44, 5.04) = 5.04  ← CLIPPED（被限制在 1.2×A）
actor_loss 貢獻 = −5.04

→ 即使策略已大幅提升此動作機率（ratio=2.01），梯度只計 ratio=1.2 的效果
→ 防止策略在一次更新中「過度轉向」
```

**情況 B：好動作（A > 0）、ratio 在正常範圍**

```
情境：同一好動作，第 3 個 epoch
      ratio = exp(0.1) = 1.10

surr1 = 1.10 × 4.2 = 4.62
surr2 = clip(1.10, 0.8, 1.2) × 4.2 = 1.10 × 4.2 = 4.62

L_CLIP = min(4.62, 4.62) = 4.62  ← 未裁剪
→ ratio 仍在 [0.8, 1.2] 內，正常梯度更新
```

**情況 C：壞動作（A < 0）、ratio 過低（已過度降低機率）**

```
情境：某步推右但桿子繼續傾倒，A = −3.5
      多個 epoch 後：log π_new = −2.2，log π_old = −0.9
      ratio = exp(−2.2 − (−0.9)) = exp(−1.3) = 0.27

surr1 = 0.27 × (−3.5) = −0.95
surr2 = clip(0.27, 0.8, 1.2) × (−3.5) = 0.8 × (−3.5) = −2.80

L_CLIP = min(−0.95, −2.80) = −2.80  ← CLIPPED
actor_loss 貢獻 = −(−2.80) = +2.80

→ 策略已大幅降低此動作機率（ratio=0.27），clip 不讓「懲罰」繼續加劇
→ 防止後期 epoch 對同一樣本過度修正
```

**三種情況對比表：**

| 情況 | ratio | A | surr1 | surr2（clip）| L_CLIP | 是否裁剪 |
|---|---|---|---|---|---|---|
| A（好動作，更新過大）| 2.01 | +4.2 | 8.44 | **5.04** | 5.04 | ✅ 是 |
| B（好動作，正常範圍）| 1.10 | +4.2 | 4.62 | 4.62 | 4.62 | ❌ 否 |
| C（壞動作，已過度懲罰）| 0.27 | −3.5 | −0.95 | **−2.80** | −2.80 | ✅ 是 |

### 三、訓練損失演化（CartPole，20K 步）

> PPO 每 n_steps=2048 步做一次批次更新（n_epochs=10，共 320 次 minibatch）

| 步數（n_steps 更新次數）| actor_loss | critic_loss | entropy | approx_kl | clip_frac |
|---|---|---|---|---|---|
| 0（第 1 次更新）| ~−0.01 | ~25–35 | ~0.69（均勻）| ~0.001 | ~0% |
| 2,048（第 1 次）| ~−0.02 | ~12–18 | ~0.65 | ~0.003 | ~2% |
| 4,096（第 2 次）| ~−0.05 | ~8–12 | ~0.60 | ~0.004 | ~5% |
| 10,240（第 5 次）| ~−0.08 | ~3–5 | ~0.55 | ~0.006 | ~8% |
| **20,480（第 10 次）**| ~−0.10 | **~0.5–1.5** | ~0.50 | ~0.007 | ~10% |

**各指標說明：**

| 指標 | 初始值 | 收斂時 | 意義 |
|---|---|---|---|
| actor_loss | ~0 | ~−0.1 | 負值：策略在改善；絕對值越大表示梯度更有效 |
| critic_loss | 25–35 | 0.5–1.5 | MSE(V(s), G_t)；隨 V 估計準確而下降 |
| entropy | 0.69（ln2）| ~0.50 | 策略從均勻分佈逐漸確定化 |
| approx_kl | ~0.001 | ~0.007 | 單次更新的 π_new vs π_old 差異（越小越穩定）|
| clip_frac | ~0% | ~10% | 約 10% 的更新被 clip 限制（正常範圍 5–20%）|

**CartPole 20K 步就達到 500 滿分的關鍵原因：**

```
1. 每批 2048 步資料重複使用 10 個 epoch（REINFORCE 每集只用 1 次）
   → 樣本效率比 REINFORCE 高 10 倍

2. clip_eps=0.2 防止「學太快又崩潰」
   → 對比 TRPO 的精確 KL 計算，計算成本低 10× 以上

3. entropy bonus（ent_coef=0.01）維持 20K 步的適度探索
   → 防止過早收斂到次優確定性策略

4. GAE(λ=0.95) 比純 TD(0) 方差低、比蒙特卡羅偏差低
   → critic 的 bootstrap 估計在短集數（CartPole 初期 ~10 步）仍有效
```

### 四、PPO vs TRPO vs REINFORCE 對比

| 面向 | REINFORCE | TRPO | PPO |
|---|---|---|---|
| 每集資料重用次數 | 1 次 | 1 次 | **10 次（n_epochs）** |
| 策略更新限制 | 無 | 精確 KL 約束（二階法）| **Clip(ratio, 0.8, 1.2)（一階法）**|
| 計算複雜度 | 低（只做梯度下降）| 高（Fisher-vector product + CG）| 低（標準梯度下降）|
| CartPole 收斂步數 | 5000 集（~55K 步），未穩定收斂 | ~350 集（~75K 步），峰值 433 | **~10 集（20K 步），滿分 500** |
| 穩定性 | 低（策略崩潰常見）| 中（KL 有時太緊導致更新停止）| **高（clip 保守，持續改善）**|

**為什麼 PPO 的 clip 比 TRPO 的 KL 約束更實用？**

```
TRPO（精確）：
  - 每次更新計算 Fisher-vector product（需要一次完整反向傳播）
  - CG 求解 10 次迭代 → 精確 KL 上界
  - 線搜尋最多 20 步 → 每次更新 = 30+ 次反向傳播
  - 保證 KL ≤ 0.01，但計算代價高

PPO（近似）：
  - 只需標準反向傳播一次
  - clip 是 KL 的代理約束（近似但實用）
  - clip_frac 顯示約 10% 更新被限制 → 等效 KL ≈ 0.003~0.009（在 TRPO 的 0.01 上界內）
  - 多個 epoch 的效果媲美 TRPO 的精確約束，且快 10×
```
