# ICM 訓練日誌（PPO + 內在好奇心模組）

環境：MountainCar-v0（稀疏獎勵，最多 200 步）｜訓練步數：1,000,000
PPO 學習率：3e-4｜折扣因子 γ：0.99｜GAE λ：0.95
內在獎勵縮放 η：1.0｜前向/逆向損失權重 β：0.2
特徵維度：256｜Rollout 長度：2,048 步

---

## 核心概念

### MountainCar-v0：稀疏獎勵的困境

```
環境設定：
  觀測值（2 維）： x（位置 -1.2 ~ 0.6）, ẋ（速度）
  動作（離散 3 維）：  0 = 向左加速, 1 = 不施力, 2 = 向右加速
  獎勵：  每步 r = -1（時間懲罰）
  終止條件：  到達右側山頂（x > 0.45）或達到 200 步上限

稀疏獎勵難點：
  - 隨機策略幾乎永遠無法到達山頂（機率 < 0.1%）
  - 代理人得到的所有回饋都是 r = -1（無論動作好壞）
  - 標準 DQN / PPO 在此環境無法學習（純外在獎勵無法提供方向訊號）
```

### ICM：用預測誤差驅動探索

```
核心思想：「代理人對自己難以預測的狀態應該感到好奇」

內在好奇心模組（ICM）包含三個子網路：
  ① FeatureEncoder φ(s)：  s → 特徵空間（256維）
  ② 逆向模型（Inverse Model）：  [φ(s), φ(s')] → 預測動作 â
  ③ 前向模型（Forward Model）：  [φ(s), a] → 預測 φ(s')

內在獎勵 r_i：
  r_i = (η / 2) × ‖φ(s') − f(φ(s), a)‖²   ← 前向模型的預測誤差

直覺解釋：
  - 代理人前往「前向模型無法預測的狀態」→ 內在獎勵高
  - 探索新狀態 → 降低預測誤差 → 內在獎勵逐漸下降
  - 自然形成「越探索越少好奇」的自主探索迴圈
```

### 結合 PPO 的訓練流程

```
外在回報 r_e = -1 / step（MountainCar 懲罰）
內在回報 r_i = η × 前向模型預測誤差

總回報 r_total = r_e + r_i

PPO 更新：每 2,048 步採集完整 rollout → 最小化 clip 損失
ICM 更新：  同時最小化逆向模型損失 + 前向模型損失
```

### 關鍵修正：特徵坍縮（Feature Collapse）

**訓練過程中發現並修正的嚴重 Bug：**

```python
# ❌ 原始程式（造成內在獎勵永遠為 0）：
return F.normalize(self.net(state), dim=-1)  # 所有 2D 輸入都對映到單位球面同一點

# ✅ 修正後：
return self.net(state)  # 保留原始特徵空間的距離關係
```

**問題根源**：MountainCar 的狀態空間只有 2 維（位置、速度）。
`F.normalize` 將所有 2D 向量投影到單位球面，
在低維空間中幾乎所有狀態都對映到相同的單位向量，
導致前向模型輸出幾乎相同，預測誤差趨近於零。

**症狀**：mean_intr_reward = 0.0000（每步），代理人完全失去好奇心驅動。
**修正效果**：mean_intr_reward 在修正後立即上升至 1.37，訓練正常進行。

同時將 η（內在獎勵縮放）從 0.01 調整為 1.0：
  確保內在獎勵（約 1.0/step）與外在獎勵（-1.0/step）處於相同量級。

### 第一個成功達頂（步數 ~800,000）

```
0–800K 步：
  - 幾乎所有集數均超時（-200 回報）
  - 代理人靠內在好奇心持續探索速度空間
  - mean_intr_reward 維持在 0.1–0.5 範圍

~800K 步：
  - 代理人首次抵達山頂（x > 0.45）
  - 此前從未到達的新狀態 → 前向模型完全無法預測 → 內在獎勵驟升
  - mean_intr_reward 重新躍升（「好奇心重燃」）
  - 此後代理人學習鞏固達頂策略
```

## 關鍵觀察

### 為何 ICM 能解決 MountainCar？

```
無 ICM 的 PPO：
  所有集數 r = -200，梯度訊號 = 0（所有動作等價）
  策略無法改善 → 永遠停留在山谷

有 ICM 的 PPO：
  每步根據新奇程度給予 r_i ∈ [0.5, 3.0]
  代理人被「驅動」向未探索的速度範圍行動
  偶然到達山頂 → 外在獎勵提升 → 鞏固成功策略
```

### 內在獎勵動態

| 訓練階段 | 內在獎勵行為 | 原因 |
|:---:|:---|:---|
| 0 ~ 500K | 穩定下降 | 代理人熟悉低速度狀態 |
| 500K ~ 800K | 維持低值 | 大多狀態已被探索 |
| 800K+ | 驟升 → 再下降 | 首次抵達山頂（全新狀態）→ 逐漸學習 |
| 1000K | 低且穩定 | 代理人已建立穩定達頂策略 |

### ICM 在稀疏獎勵問題中的角色

```
問題：外在獎勵完全無法提供方向性的學習訊號
解法：ICM 用「自監督預測誤差」創造密集的內在獎勵

關鍵成功要素：
  1. η 足夠大（≥ 1.0）：內在獎勵能與外在獎勵抗衡
  2. 特徵空間設計：不能使用 F.normalize（低維空間特徵坍縮）
  3. 足夠的訓練步數（100K 遠遠不夠，需要 800K+ 才能首次達頂）

延伸閱讀：
  - RND（Random Network Distillation, 2018）：更簡單的內在獎勵機制
  - Count-based exploration：訪問次數的內在獎勵（表格型問題的精確版本）
  - Go-Explore：直接重置到探索的起點（解決探索問題的不同方法）
```
