## 觀察

- **評論家損失**：全程穩定在 66–72，`critic_loss ≈ bellman + cql_alpha × cql_penalty ≈ 4.0 + 5×12.7 ≈ 67`，數字吻合。
- **CQL 懲罰項**：14.1 → 12.6（-11%），表示策略動作與資料集動作 Q 值差距持續縮小。
- **演員損失關鍵轉折**：步數 100k 前為負（-23→-3，SAC 正常行為），100k 後**轉為正值並持續上升**（+1.6→+15.8）。
  - 負值：Q(s,a) 大於 α·log_π，策略積極學習提高 Q 值動作
  - 正值：保守懲罰項開始主導，策略被推向更保守（更接近資料集分佈）的動作
  - 這是 CQL 在長訓練後的典型行為：保守性越來越強，抑制了策略的激進探索
- **評估回報趨勢**：50k 峰值後緩慢下降（1704→1189），與演員損失轉正時機吻合。
  說明 CQL 的保守懲罰在後期過度壓制了策略，是 cql_alpha=5.0 偏高的副作用。
- **Alpha（SAC 熵溫度）**：0.0083 → 0.0133，自動調整上升，顯示策略在後期趨向更保守（更低熵）。

## CQL 核心原理

```
離線 RL 的核心挑戰：分佈外（OOD）動作高估問題
  標準 Q 學習會對 OOD 動作賦予過高 Q 值 → 策略選擇未見過的動作 → 崩潰

CQL 的解法：在 Bellman 更新中加入保守懲罰項

L_CQL = L_Bellman + alpha * CQL_penalty
CQL_penalty = E_{a~π}[Q(s,a)] - E_{a~D}[Q(s,a)]
              ^^^^^^^^^^^^^^^^    ^^^^^^^^^^^^^^^^
              壓低（OOD 動作）    推高（資料集動作）

效果：Q 函式成為真實價值的下界，僅讓資料集中的動作保持高 Q 值。
```
