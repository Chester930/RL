## 觀察

- **評論家損失**：從 74.2 下降至 67.2，趨勢平穩，表示 Bellman 備份正常收斂。
  `critic_loss = bellman_loss + cql_alpha * cql_penalty ≈ 4.0 + 5×12.7 ≈ 67`，數字吻合。
- **CQL 懲罰項**：從 14.1 穩定降至 12.7，表示策略動作的 Q 值與資料集動作 Q 值的差距
  正在縮小（策略被拉向資料集分佈）。
- **演員損失**：從 -23.9 收斂至 -3.1，負號為 SAC 最大化 Q - α·log_π 的正常行為；
  絕對值縮小表示策略趨於穩定。
- **Alpha 自動調整**：0.051 → 0.011，SAC 的自動熵調整快速將溫度調低，
  策略很快趨於確定性（目標熵 = -action_dim = -6）。
- **評估回報**：步數 25k 時評估回報異常為 -239（策略初期高度保守），
  步數 50k 後躍升至 1700+。在 HalfCheetah 環境中，1700 屬於較高表現，
  即使使用隨機資料集，CQL 的保守策略提取仍然能學到有效的前進行為。
  這展示了 CQL 在有限質量資料上的隱式行為提取能力。
- **無 Checkpoint**：進程在步數 100k 附近中斷，步數 100k 的 checkpoint 未能成功儲存。

## CQL 核心原理

```
離線 RL 的核心挑戰：分佈外（OOD）動作高估問題
  標準 Q 學習會對 OOD 動作賦予過高 Q 值 → 策略選擇未見過的動作 → 崩潰

CQL 的解法：在 Bellman 更新中加入保守懲罰項

L_CQL = L_Bellman + alpha * CQL_penalty
CQL_penalty = E_{a~π}[Q(s,a)] - E_{a~D}[Q(s,a)]
              ^^^^^^^^^^^^^^^^    ^^^^^^^^^^^^^^^^
              壓低（OOD 動作）    推高（資料集動作）

效果：Q 函式成為真實價值的下界，僅讓資料集中的動作保持高 Q 值。
```
