## 5. 演演算法二：策略疊代 (Policy Iteration)

### 白話說明

> 先定一套走法，評估這套走法能得幾分，再根據分數改進走法，重複直到走法不再改變。

**第一步：隨便定一套初始走法（例如：所有格子都往左走）**
=======================

=== File: .\01_Tabular_Basics\1950s_DP\README.md ===
**第二步（評估）：按照這套走法走，計算每個格子的期望分數**

注意：這裡不試其他方向，只算「走法規定的那個方向」會得幾分，一直算到分數穩定為止。

「全往左」是爛走法，所以算出來的分數很低，但這是如實反映這套走法的結果。

**第三步（改進）：拿著算好的分數，對每個格子問「換個方向會更好嗎？」**
=======================

=== File: .\01_Tabular_Basics\1950s_DP\README.md ===
**第四步：改完走法之後，回到第二步重新評估新走法**

**直到某輪改進後，所有格子的走法都沒有改變 → 這套走法已經是最佳的了，停止**

### 為什麼只需要 7 輪？

每輪的改進都是「有完整資訊支撐」的大幅更新——先把當前走法徹底評估清楚，再一口氣把所有格子能改的都改掉。相比 VI 的每次只微調一點點，PI 每輪改進的幅度大很多，所以輪數少得多。

### 逐步運算記錄

執行 `python train.py` 後，同目錄會自動產生 `training_log.md`。

裡面記錄 PI 每一輪（共 7 輪）的完整資料：評估後的 V^π 表格、哪些格子改變了方向、改變前後的箭頭、以及改進後的完整策略表格。

**規律：** 每輪評估都比上一輪更精確（因為走法更好了），改進時能做出更大的調整。PI 只需要 7 輪就能找到最佳走法，因為每輪都是「充分評估後的有資訊改進」，而不是像 VI 那樣每次只微調一點點。

### 為什麼每次改進都不會變差？

因為我們是「知道了現在走法的分數之後，才換成更好的方向」。換的條件是「新方向分數 ≥ 舊方向分數」，所以每次只會持平或變好，絕對不會變差。

有限的格子數量 × 每次只進不退 → 一定會在有限輪數內找到最佳走法。

## 6. 實際執行

### 安裝與執行
=======================

=== File: .\01_Tabular_Basics\1950s_DP\README.md ===
FrozenLake 的規則可以查表，所以 DP 能用。但現實世界呢？

- 下圍棋：每一步之後的局面機率是多少？→ 根本算不出來
- 控制機械手臂：物理碰撞的精確機率？→ 沒有公式
- 股票交易：明天漲跌的機率？→ 沒人知道

→ **這催生了「不需要知道規則」的方法：蒙特卡羅（MC）直接讓 Agent 去玩，從實際結果裡學。**

### 問題二：格子數一多，表格就爆了

本例 16 個格子 × 4 個方向 = 64 格，小事一樁。

但如果是 Atari 電玩遊戲，畫面有 84×84 個畫素，每個畫素有 256 種顏色——狀態數量是天文數字，根本存不下這張表。

→ **這催生了深度強化學習（DQN），用神經網路取代表格，不管狀態有多複雜都能處理。**

優勢 (Advantage):  A_t = G_t - V(s_t)    其中 G_t = r_t + gamma*r_{t+1} + ... + gamma^{n-1}*r_{t+n} + gamma^n * V(s_{t+n})

評論家損失 (Critic loss):  L_critic = (V(s_t) - G_t)^2
熵損失 (Entropy):         L_entropy = -H(pi(s_t))   [鼓勵探索]

總損失:  L = L_actor + c_v * L_critic - c_e * L_entropy

=== File: .\03_Policy_Gradient\2015_TRPO\README.md ===
1. 在舊策略 pi_old 下進行資料取樣 (Rollouts)
2. 計算優勢函式 A_t (通常使用 GAE)
3. 計算策略梯度 g = grad L(theta_old)
4. 求解 Hx = g 以獲得自然梯度方向 x
   (H = Fisher 訊息矩陣 = E[grad log pi * grad log pi^T])
5. 計算滿足 KL 約束的最大步長：
   s = sqrt(2*delta / x^T H x) * x
6. 使用回溯線性搜尋 (Backtracking line search) 尋找滿足 KL 的最大實際步長
7. 使用梯度下降更新評論家 (Critic) 網路

=== File: .\06_Advanced_Specialized\2017_ICM\README.md ===
1. 特徵編碼器 (Feature Encoder): s -> phi(s)
   透過逆向模型訓練，僅捕捉環境中「可控」的部分。

   預測下一個狀態的特徵。預測誤差越高，代表該區域越新奇，給予的好奇心獎勵就越高。

=== File: .\07_Modern_RLHF\2023_DPO\README.md ===
最優 RLHF 策略具有以下解析解 (Closed form)：
    pi*(y|x) ∝ pi_ref(y|x) * exp(r(x,y) / beta)

重新排列可得： r(x,y) = beta * log(pi*(y|x) / pi_ref(y|x)) + Z(x)

將此獎勵項代入 Bradley-Terry 偏好模型：
    p(y_w > y_l | x) = σ(r(x, y_w) - r(x, y_l))
                    = σ(beta * [log(pi(y_w|x)/pi_ref(y_w|x))
                               - log(pi(y_l|x)/pi_ref(y_l|x))])

DPO 損失函式： L = -E[ log σ(beta * (log_ratio_w - log_ratio_l)) ]
=======================

    A_i = (r_i - mean(r)) / std(r)

原始問題（硬約束）：
  max E[Σ r_t]  s.t.  E[Σ c_t] ≤ d

轉換（拉格朗日鬆弛）：
  L(π, λ) = E[Σ r_t] - λ × (E[Σ c_t] - d)

  λ（拉格朗日乘數）自動調整：
    違規太多 → λ ↑ → 安全代價懲罰加重 → 策略更保守
    違規太少 → λ ↓ → 懲罰減弱 → 策略更積極
