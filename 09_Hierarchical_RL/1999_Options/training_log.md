# Options 訓練日誌

> 環境：FourRooms GridWorld（13×13，4 個房間，4 個走廊 option）  
> 任務：從左上角（1,1）到右下角（9,11）  
> 訓練規模：5000 集，同步訓練 Options + Flat Q-learning 對照組

---

## 演算法特有指標

| 指標 | 說明 |
|---|---|
| Options 成功率 | 在評估集數中到達目標的比例 |
| Flat Q 成功率 | 同條件下純 Q-learning 的成功率（對照） |
| Options 平均集數獎勵 | 每集累積 env 獎勵（到達 +1，每步 -0.001） |
| Flat Q 平均集數獎勵 | 同上（對照） |

---

## 訓練結果

| 集數 | Options 成功率 | Flat Q 成功率 | 備註 |
|---|---|---|---|
| 0 | ~0% | ~0% | 隨機探索 |
| 500 | 84.0% | 100.0% | Options 已能走到目標 |
| 1000 | 80.0% | 100.0% | Flat Q 穩定收斂 |
| 2000 | 82.0% | 100.0% | Options 穩定在 80-84% |
| 5000（最終） | 84.0% | 100.0% | Options 最佳 84%，Flat Q 100% |

**最佳 Options 成功率**：84.0%（集數 500）

---

## Options 優勢分析

**Options 首次到達目標的集數**：極早（集數 200 獎勵已達 0.912，代表大多數集數成功抵達）

**收斂速度**（達到 80% 成功率所需的集數）：Options ~500 集；Flat Q ~500 集（相近）

**最終成功率差距**：Flat Q 100% vs Options 84%，Flat Q 表現更穩定

**觀察**：
- 兩者均在 500 集前快速學習，FourRooms 對 tabular Q-learning 並不困難
- Flat Q 最終 100% 成功；Options 穩定在 80~84%
- Options 的不確定性來自高層策略（eps_hi=0.3）在 eval 下雖已關閉，但 option 切換序列可能產生次優路徑
- 獎勵曲線兩者接近（~0.97~0.98），顯示兩者均高效到達目標（步數少）

## 教學重點

Options 的核心優勢：時間抽象讓 agent 不需要規劃數百步原始動作，
而是選擇「走向哪個走廊」這樣的高層目標。

```
高層策略（每次 option 選擇）：
  Q_hi(s, option)：應該去哪個走廊？

低層策略（每步動作）：
  Q_lo[option](s, action)：在走向目標走廊的過程中，每步怎麼走？

Option 終止條件：到達目標走廊 → 回到高層選下一個 option
```

本實驗結論：在 13×13 FourRooms 中，Flat Q 因狀態空間小（~130 格）可完全收斂至 100%。
Options 框架的優勢在更大、更複雜的環境中更顯著——此處展示了階層式分解的可行性與語意性。
