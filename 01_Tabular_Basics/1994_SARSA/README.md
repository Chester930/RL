# SARSA（同策略 TD 控制）

> 年代：1994｜方法：無模型、同策略、每步線上更新｜環境：FrozenLake-v1

---

## 1. SARSA 是什麼？

想像你在 FrozenLake 裡邊走邊學，每走一步就立刻更新你的判斷。

**TD(0) 的做法**：每步更新，但只估計「狀態價值」V(s)——不管你選了哪個動作。

**Q-Learning 的做法**：每步更新，更新目標是「下一格所有動作中最大的 Q 值」——不管你實際選了哪一步，目標永遠是「最優那個」。

**SARSA 的做法**：每步更新，更新目標是「下一格**你實際選的那個動作**的 Q 值」——學的是你**現在這個策略**（包含探索）實際走下去的期望。

**這叫「同策略（On-policy）」：用哪個策略探索，就學哪個策略的價值——探索行為直接影響你學到的 Q 值。**

| 比較專案 | MC | Q-Learning（異策略）| SARSA（同策略）|
|:---|:---:|:---:|:---:|
| 需要走完整集？ | ✅ 必須 | ❌ 不需要 | ❌ 不需要 |
| 更新目標 | 實際回報 G | max Q(s′, 所有 a′) | Q(s′, **實際選的** a′) |
| 學習的策略 | 當前策略 | 最優策略 Q* | 當前 ε-greedy 策略 |
| 探索會影響 Q？ | 會 | **不會** | 會 |

## 2. FrozenLake 環境

4 × 4 的冰湖，共 16 個格子：

```
 S  F  F  F       0   1   2   3
 F  H  F  H  →   4  [5]  6  [7]
 F  F  F  H       8   9  10  [11]
 H  F  F  G      [12] 13  14   15
```

- **S（0）**：起點
- **F**：普通冰面，可行走
- **H（5, 7, 11, 12）**：洞口，掉進去就結束這集，獎勵 0
- **G（15）**：終點，抵達得 1 分

**冰面會滑！** 你選擇往「下」走，實際有：
- 1/3 機率真的往下
- 1/3 機率往右滑
- 1/3 機率往左滑

所以就算走對方向，也不保證走到想去的格子。

## 3. 三個核心概念

### 3-1. 集數（Episode）

從起點出發，走到終點或掉進洞口，這一整段路叫做「一集」。  
SARSA **不需要**走完整集，每走一步就立刻更新 Q 值。

```
一集範例：
起點(0) → 格子4 → 格子8 → 格子9 → 格子13 → 格子14 → 終點(15) ✓
```

### 3-2. 同策略 TD 目標（On-policy Target）

SARSA 每一步的更新目標是：

$$\text{Target} = r + \gamma Q(s', a')$$

- $r$：這一步的即時獎勵
- $\gamma = 0.99$：折扣因子
- $Q(s', a')$：下一格「實際選的那個動作」的 Q 值
- $a'$：由 $\epsilon$-greedy 策略選出，不一定是最大 Q 的那個

**為什麼用實際選的 a′，而不是 max？**  
SARSA 想學的是「按照目前這個策略（包含 ε 的隨機探索）走下去，真實的期望報酬是多少」。如果你常常會因為 ε 而隨機走到差的動作，Q 值就應該反映這一點——這讓 SARSA 在訓練期間對危險的格子更保守，因為探索到壞動作的代價已經被計入 Q 值。

### 3-3. 動作價值 Q(s, a)

「在格子 s 採取動作 a，**按照目前的 ε-greedy 策略**繼續走，預期能得到多少折扣後的總獎勵。」

$$Q(s, a) \leftarrow Q(s, a) + \alpha \delta$$

- $\delta = \text{Target} - Q(s, a)$ （TD 誤差）
- $\alpha = 0.1$ （學習率）

其中：
- $\delta > 0$：目標比目前估計高，調高 $Q(s,a)$
- $\delta < 0$：目標比目前估計低，調低 $Q(s,a)$
- $\delta = 0$：估計已精準，保持不變

**與 Q-Learning 的關鍵差異**：

- **Q-Learning**：
  $$\delta = r + \gamma \max_{a'} Q(s', a') - Q(s, a) \quad \text{（TD 誤差不受 } \epsilon \text{ 隨機探索影響）}$$
- **SARSA**：
  $$\delta = r + \gamma Q(s', a') - Q(s, a) \quad \text{（TD 誤差隨 } \epsilon \text{ 探索選擇的 } a' \text{ 而變）}$$

## 4. ε-greedy 探索策略

**問題**：每次都選 Q 最高的動作，可能永遠不去探索其他路線，錯過更好的走法。

**解法**：大多數時候選最好的，偶爾隨機探索。

```
以機率 ε   → 隨機選一個動作（探索）
以機率 1-ε → 選 Q 最高的動作（利用）
```

**ε 衰減**：訓練初期 ε = 1.0（幾乎全部隨機探索），隨著訓練進行逐漸降低，到 ε = 0.01（99% 按最佳策略走）。

| 訓練階段 | ε 約值 | 行為 |
|:---:|:---:|:---|
| 剛開始 | 1.00 | 完全隨機，像個迷路的觀光客 |
| 中間段 | 0.14 | 主要按最佳走法，偶爾隨機探索 |
| 末期 | 0.02 | 幾乎全按最佳走法，很少嘗試 |

**SARSA 的同策略效應**：訓練前期 ε 很高，a′ 常常是隨機選的差動作，導致 Q(s′, a′) 偏低——Q 值因此被壓低，讓策略自然迴避危險格子附近的路線。等 ε 衰減後，Q 值才逐漸逼近最優。

## 5. SARSA 演演算法步驟

```
重複以下流程 N 次（N = 20,000 集）：

步驟一：觀察初始格子 s，用 ε-greedy 選動作 a

步驟二：走一步，觀察 r 和下一格 s′

步驟三：用 ε-greedy 選下一動作 a′（必須在更新前選！）

步驟四：計算同策略 TD 目標
         $$\text{Target} = r + \gamma Q(s', a')$$
         （集數結束時：$$\text{Target} = r$$）

步驟五：計算 TD 誤差
         $$\delta = \text{Target} - Q(s, a)$$

步驟六：更新 Q(s, a)
         $$Q(s, a) \leftarrow Q(s, a) + \alpha \delta$$

步驟七：s ← s′，a ← a′（直接複用剛才選的 a′，不重新選）

步驟八：若集數未結束，回到步驟二；ε 稍微衰減（× 0.9998）
```

**為什麼必須在更新前選 a′？**  
SARSA 的更新目標是 `Q(s′, a′)`，必須知道 a′ 才能計算 δ。而且 a′ 選好之後，下一步就直接拿來用（步驟七 `a ← a′`），這樣保證「選的策略」和「學的策略」完全一致——這正是同策略的定義。

**這就是 SARSA 名稱的由來**：每次更新用到的五個元素依序是 **S**tate → **A**ction → **R**eward → next **S**tate → next **A**ction。

**SARSA vs Q-Learning 更新公式對比：**

- **Q-Learning（異策略）**：
  $$Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \right]$$
- **SARSA（同策略）**：
  $$Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma Q(s', a') - Q(s, a) \right] \quad \text{（其中 } a' \text{ 由 } \epsilon\text{-greedy 實際選出）}$$

## 7. 訓練結果

訓練 20,000 集的典型進度：

| 集數 | 平均成功率 | ε 約值 | 說明 |
|:---:|:---:|:---:|:---|
| 5,000 | ~5% | 0.37 | 幾乎全探索，很少成功 |
| 10,000 | ~20% | 0.14 | 開始學到靠近終點的路線 |
| 15,000 | ~43% | 0.05 | 進步明顯，策略趨於穩定 |
| 20,000 | ~58% | 0.02 | 最終策略趨於穩定 |

**為什麼最高只有 57% 左右？**  
FrozenLake 的冰面很滑，就算走最佳路線，仍然有隨機滑動。理論上最佳策略的成功率約為 74%——需要更多集數和更細緻的探索才能接近。

## 8. SARSA vs Q-Learning 比較

| 比較專案 | SARSA（同策略）| Q-Learning（異策略）|
|:---|:---|:---|
| **更新目標** | r + γ × Q(s′, **實際選的** a′) | r + γ × **max** Q(s′, a′) |
| **學習的策略** | 當前 ε-greedy 策略 | 最優策略 Q* |
| **探索影響** | ε 越大，Q 越保守 | ε 不影響學到的 Q 值 |
| **懸崖邊** | 繞遠路，更安全 | 走捷徑，可能掉下去 |
| **收斂目標** | Q^π（當前策略的 Q）| Q*（最優）|
| **適用場景** | 線上部署、安全敏感 | 離線學習、重放緩衝 |

**SARSA 的最大優勢**：在訓練過程中就已經安全。因為 Q 值反映的是「帶著探索行為實際走下去」的期望，策略天然傾向迴避探索時容易出事的危險區域。這讓 SARSA 特別適合「訓練期間就要部署、不能出錯」的場景——例如機器人現場學習，不能在真實環境裡一邊探索一邊掉進洞裡。

## 9. SARSA 的限制（為什麼接下來需要函式近似？）

SARSA 有兩個根本弱點：

**弱點一：表格無法擴充**  
Q 表大小 = 狀態數 × 動作數。FrozenLake 只有 16×4=64 個值，但真實世界的狀態可能是幾億個（棋盤）甚至連續的（機器人關節角度、遊戲畫素畫面）——根本存不進記憶體。

**弱點二：收斂到 Q^π 而非 Q***  
SARSA 學到的是「帶著 ε 探索的策略的 Q 值」——只要 ε > 0，Q^π 就偏低於真正的最優 Q*。ε 衰減後可以逼近，但永遠有差距；ε 衰減太快則失去探索，容易陷在區域性次優。Q-Learning 天生跳過這個問題（目標直接取 max），但也因此犧牲了訓練期間的安全性。

**函式近似（Function Approximation）**的解法：  
不用表格存 Q(s,a)，改用神經網路來近似。SARSA 的同策略更新規則完全不變——只是把 `Q[state, action]` 換成神經網路的輸出。這讓 SARSA 天然適合「Actor-Critic」架構中的線上學習，成為後來 A3C、PPO 等演算法的理論前身。

## 10. 結語

SARSA 是 1994 年 Rummery & Niranjan 提出、Sutton 1996 年正式命名的演算法，核心洞察只有一句話：**用實際走的 a′ 當目標，學到的就是你真實在走的那個策略的價值。**

它在 Q-Learning 的框架上只改了一個字——把 `max` 換成「實際選的」——但這一個字讓學習目標從「理想最優」變成「當前真實」。這個差異在危險環境裡至關重要：SARSA 知道自己有時會不小心走錯，所以提前把這個風險算進 Q 值，選擇更謹慎的路線。

從 SARSA 到後來的 Actor-Critic，再到現代的 PPO，同策略的核心邏輯從未改變：**你用什麼策略收集資料，就學什麼策略的價值。** 理解 SARSA，就理解了現代深度強化學習中「線上學習」分支的根基。
