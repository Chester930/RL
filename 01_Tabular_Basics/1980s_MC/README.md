# 蒙特卡羅方法 (Monte Carlo, MC)

> 年代：1980s｜方法：無模型、從完整集數學習｜環境：FrozenLake-v1

---

## 1. 蒙特卡羅是什麼？

想像你要學會走一個你從沒去過的迷宮。

**動態規劃（DP）的做法**：先拿到完整的迷宮地圖，知道每一格走哪個方向、有多少機率滑到哪裡，然後坐在桌子前用算術把答案推出來。

**蒙特卡羅（MC）的做法**：不看地圖，直接進去走。走一遍，記下整路拿到了多少分；走完一整圈回到起點後，回頭看這趟路，更新對每個地方的判斷。然後再走下一圈，重複幾萬次，慢慢學會哪條路最好走。

**關鍵差異**

| 比較專案 | DP（動態規劃）| MC（蒙特卡羅）|
|:---|:---:|:---:|
| 需要地圖？（轉移機率 P）| ✅ 一定要 | ❌ 不需要 |
| 怎麼學習 | 用公式直接算 | 真實走過後再學 |
| 學習時機 | 每算一步就更新 | 走完整集才更新 |
| 現實中能用嗎 | 只能用在規則完全已知的環境 | 可用在真實世界 |

## 2. FrozenLake 環境

4 × 4 的冰湖，共 16 個格子：

```
 S  F  F  F       0   1   2   3
 F  H  F  H  →   4  [5]  6  [7]
 F  F  F  H       8   9  10  [11]
 H  F  F  G      [12] 13  14   15
```

- **S（0）**：起點
- **F**：普通冰面，可行走
- **H（5, 7, 11, 12）**：洞口，掉進去就結束這集，獎勵 0
- **G（15）**：終點，抵達得 1 分

**冰面會滑！** 你選擇往「下」走，實際有：
- 1/3 機率真的往下
- 1/3 機率往右滑
- 1/3 機率往左滑

所以就算走對方向，也不保證走到想去的格子。

---

## 3. 三個核心概念

### 3-1. 集數（Episode）

從起點出發，走到終點或掉進洞口，這一整段路叫做「一集」。  
MC 必須走**完整一集**後，才能回頭學習。

```
一集範例：
起點(0) → 格子4 → 格子8 → 格子9 → 格子13 → 格子14 → 終點(15) ✓
```

### 3-2. 回報 G（Return）

走完一集後，從某個步驟 t 往後看，把剩下的獎勵全部加起來（但愈遠的獎勵打折）：

```
G_t = r_t + γ × r_{t+1} + γ² × r_{t+2} + ...
    = r_t + γ × G_{t+1}          ← 倒序計算公式

  r_t ：步驟 t 的即時獎勵
  γ   ：折扣因子（0.99 代表下一步的獎勵只算 99%）
```

**為什麼要打折？**  
同樣是拿 1 分，現在拿比等 10 步後再拿更好。折扣因子 γ 體現了「早點拿到比晚拿到好」的概念。

### 3-3. 動作價值 Q(s, a)

「在格子 s 採取動作 a，按照目前的走法繼續走，平均能得到幾分。」

MC 用多次集數的 G 值取平均來估計 Q：

```
Q(s, a) = 所有這個 (格子, 動作) 組合的 G 值平均
```

走的集數越多，平均越穩定，Q 估計越準。

---

## 4. ε-greedy 探索策略

**問題**：每次都選 Q 最高的動作，可能永遠不去探索其他路線，錯過更好的走法。

**解法**：大多數時候選最好的，偶爾隨機探索。

```
以機率 ε  → 隨機選一個動作（探索）
以機率 1-ε → 選 Q 最高的動作（利用）
```

**ε 衰減**：訓練初期 ε = 1.0（幾乎全部隨機探索），隨著訓練進行逐漸降低，到 ε = 0.01（99% 按最佳策略走）。

| 訓練階段 | ε 約值 | 行為 |
|:---:|:---:|:---|
| 剛開始 | 1.00 | 完全隨機，像個迷路的觀光客 |
| 中間段 | 0.37 | 大約各半，邊探索邊利用 |
| 末期 | 0.01 | 幾乎全按最佳走法，偶爾嘗試 |

## 5. First-Visit MC 演演算法步驟

```
重複以下流程 N 次（N = 50,000 集）：

步驟一：用 ε-greedy 走完整一集
         記錄：(格子, 動作, 獎勵) × 全程步數

步驟二：從最後一步往前，倒算每步的 G 值
         G_t = r_t + γ × G_{t+1}
         （最後一步的 G_{t+1} = 0）

步驟三：對每個 (格子, 動作) 組合，只取這集中第一次出現的 G 來更新
         N(s,a) ← N(s,a) + 1
         Q(s,a) ← Q(s,a) + (1/N) × (G - Q(s,a))

步驟四：ε 稍微衰減（× 0.9999）

步驟五：回到步驟一，繼續下一集
```

**為什麼「首次存取」？**  
同一集數中，若格子 s 被走過兩次，後面那次的 G 值已經包含了前面那次的路徑——重複計算會讓估計有偏差。只取第一次，確保每次的 G 是獨立的。

**增量平均的意義**：

```
Q(s,a) ← Q(s,a) + (1/N) × (G - Q(s,a))

N=1 時：Q 直接等於這次的 G（沒有歷史可參考）
N=10 時：新 G 隻影響 Q 的 1/10（過去 9 次也很重要）
N=1000 時：每次更新幅度很小，Q 已很穩定
```

## 7. 訓練結果

訓練 50,000 集的典型進度：

| 集數 | 平均成功率 | ε 約值 | 說明 |
|:---:|:---:|:---:|:---|
| 5,000 | ~3% | 0.61 | 幾乎全探索，很少成功 |
| 10,000 | ~7% | 0.37 | 開始學到一些有用路線 |
| 20,000 | ~22% | 0.14 | 進步明顯 |
| 30,000 | ~37% | 0.05 | 逐漸收斂 |
| 50,000 | ~47% | 0.01 | 最終策略趨於穩定 |

**為什麼最高只有 47% 左右？**  
FrozenLake 的冰面很滑，就算走最佳路線，仍然有隨機滑動。理論上最佳策略的成功率約為 74%——需要更多集數和更細緻的探索才能接近。

## 8. MC vs DP 比較

| 比較專案 | DP（動態規劃）| MC（蒙特卡羅）|
|:---|:---|:---|
| **需要環境模型** | ✅ 需要完整的 P(s'\|s,a) | ❌ 不需要 |
| **學習來源** | 用 Bellman 方程計算 | 從真實集數的 G 學 |
| **更新時機** | 每掃一遍所有格子 | 每走完一整集 |
| **估計準確度** | 精確（用公式直接推） | 近似（需要大量樣本） |
| **適用場景** | 規則完全已知的小型環境 | 只能玩遊戲的真實場景 |
| **對長集數** | 影響不大 | 必須等整集結束才更新，很慢 |

**MC 的最大優勢**：不需要知道環境的轉移機率。只要能和環境互動，就能學習——這在真實世界（例如機器人、遊戲 AI）中非常重要。

---

## 9. MC 的限制（為什麼接下來需要 TD？）

MC 有兩個根本弱點：

**弱點一：必須走完整集才能學**  
如果一集有 1000 步，每次都得走完才能更新 Q——速度很慢。如果集數沒有終點（比如持續執行的機器人），MC 根本無法使用。

**弱點二：G 的估計有很大的隨機性**  
G 是整集所有隨機結果的總和。冰面的滑動每次都不同，導致同樣的 (格子, 動作) 每次 G 值差異很大，需要極大量集數才能取到穩定的平均。

**TD（時序差分，Temporal Difference）**的解法：  
不等整集結束，走一步就更新一次——結合了 DP「每步更新」和 MC「不需要地圖」的優點。這就是下一個要學的方法（Q-learning、SARSA）。

---

## 10. 結語

蒙特卡羅方法是從「需要完整地圖」走向「從真實經驗學習」的第一步。

它的核心精神很直觀：**做很多次實驗，取平均，答案就出來了**——就像賭場預估莊家優勢，不需要算數學，只要開幾千次賭局，統計一下就知道。

MC 的限制（必須等整集完成）催生了 TD 方法；TD 的成功最終演化出 DQN、PPO 等現代深度強化學習演演算法。理解 MC，就理解了現代 RL 的一半起源。
