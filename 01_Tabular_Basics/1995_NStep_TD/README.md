# N-步時序差分（N-Step TD）

> 年代：1988/1995｜方法：無模型、同策略、n 步批次更新｜環境：FrozenLake-v1

---

## 1. FrozenLake-v1 環境

4 × 4 的冰湖，共 16 個格子：

```
 S  F  F  F       0   1   2   3
 F  H  F  H  →   4  [5]  6  [7]
 F  F  F  H       8   9  10  [11]
 H  F  F  G      [12] 13  14   15
```

- **S（0）**：起點
- **F**：普通冰面，可行走
- **H（5, 7, 11, 12）**：洞口，掉進去就結束這集，獎勵 0
- **G（15）**：終點，抵達得 1 分

**冰面會滑！** 你選擇往「下」走，實際有：
- 1/3 機率真的往下
- 1/3 機率往右滑
- 1/3 機率往左滑

| 事件 | 獎勵 r |
|:---|:---:|
| 普通移動（F 格之間） | 0 |
| 成功抵達終點 G | **+1** |
| 掉入洞口 H | 0（直接結束）|

---

## 2. N-步 TD 是什麼？（直覺說明）

### 問題：TD(0) 的信用傳播太慢

FrozenLake 只有到達終點才有 +1 獎勵，沿途全是 0。

**TD(0)（每走一步就立刻更新）** 的問題：
- 你走一步，獎勵 r = 0，下一格 Q 值也幾乎是 0 → 等於什麼都沒學到
- 終點獎勵 +1 每集只能往前傳「一格」
- 要讓起點附近的格子學到有用的資訊，需要走很多很多集

### N-步 TD 的解法：記住最近 n 步，一起算

**不急著每步更新，先把最近 4 步記下來，等到第 4 步完成後，把這 4 步的獎勵全部加起來，才更新第 1 步的 Q 值。**

```
n=1（TD(0)）：走一步 → 立刻更新，只看 1 步後的估計
n=4（本次）：走 4 步 → 才更新，把 4 步真實獎勵加起來再估計
n=∞（MC）  ：走完整集 → 才更新，用整集所有獎勵
```

**比喻**：
- TD(0) 像「吃第一口就評價一道菜」——快，但資訊少
- N-步 TD 像「吃完 4 道菜再評分」——多等幾步，判斷更準確
- MC 像「吃完整桌席才打分」——最準，但要等到最後

## 3. 4 步滑動視窗（如何運作）

### 核心機制

n=4 代表：程式維護一個「最近 4 步的記憶盒」，每走一步就把最舊的一筆拿去更新，加入最新的一步。

```
步驟：  1    2    3    4    5    6    7    8
格子：  s1   s2   s3   s4   s5   s6   s7   s8
獎勵：  r1   r2   r3   r4   r5   r6   r7   r8

4步視窗的滑動：

步驟 4 完成時 → 視窗 [1,2,3,4] → 計算 G，更新 Q(s1, a1)
步驟 5 完成時 → 視窗 [2,3,4,5] → 計算 G，更新 Q(s2, a2)
步驟 6 完成時 → 視窗 [3,4,5,6] → 計算 G，更新 Q(s3, a3)
步驟 7 完成時 → 視窗 [4,5,6,7] → 計算 G，更新 Q(s4, a4)
...
```

### 獎勵訊號如何傳播

以 FrozenLake 為例（終點在步驟 13，r=+1）：

```
步驟：  10   11   12   13（終點 r=+1）
         ↑────────────────↑
   更新 Q(步驟10)   ←   r=+1 往前傳了 4 步！
```

相比 TD(0) 每集只能傳 1 步，**n=4 讓同一集的獎勵訊號傳播 4 倍遠**——加速學習。

### 集數結束時的 Flush

集數結束時，記憶盒裡還剩 3 筆沒有更新，必須一次清空（Flush）：

```
集數結束：步驟 13 是最後一步

還未更新的：步驟 11、12、13
→ 分別用「從那步到集數結束的所有獎勵」計算 G
→ 沒有後續狀態可估計，直接用真實獎勵加總
```

## 4. n 的選擇：平衡速度與準確

| n 值 | 學習速度 | 準確性 | 適用場景 |
|:---:|:---:|:---:|:---|
| **1**（TD(0)）| 快（每步立刻更新）| 靠估計，可能不準 | 需要即時反應 |
| **4**（本次）| 中 | 中 | 大多數情況的預設值 |
| **8**～**16** | 慢 | 較準 | 獎勵密集的任務 |
| **∞**（MC）| 最慢（等整集）| 最準（無估計誤差）| 集數不長的任務 |

> **實務建議**：n = 4 ～ 8 通常效果最好。n 太小像 TD(0)（估計不夠準），n 太大像 MC（等太久才更新）。

---

## 5. 核心思想（進階）

N-步 TD 提供了 **TD(0) 與蒙特卡羅之間的連續光譜**：

| n 值 | 名稱 | 估計誤差（偏差）| 結果波動（變異數）| 說明 |
|:---:|:---:|:---:|:---:|:---|
| **1** | TD(0) | 最高 | 最低 | 單步更新，依賴當前 Q 值估計 |
| **n** | N-步 TD | 中 | 中 | 前看 n 步，平衡估計誤差與波動 |
| **∞** | 蒙特卡羅 | 無 | 最高 | 使用整集真實回報，不依賴估計 |

---

## 6. N-步回報公式

```
G_{t:t+n} = r_{t+1} + γ × r_{t+2} + ... + γ^{n-1} × r_{t+n}
           + γ^n × Q(s_{t+n}, a_{t+n})    ← 第 n 步後用 Q 估計

Q(s_t, a_t) ← Q(s_t, a_t) + α × [G_{t:t+n} − Q(s_t, a_t)]
```

集數結束時（Flush），後面沒有狀態可估計：

```
G = 剩餘步驟的折扣獎勵加總（無估計項）
```

## 8. 訓練結果

訓練 20,000 集（n=4）的典型進度：

| 集數 | 成功率 | ε 約值 |
|:---:|:---:|:---:|
| 5,000 | ~3% | 0.37 |
| 10,000 | ~10% | 0.14 |
| 15,000 | ~20% | 0.05 |
| 20,000 | ~29% | 0.02 |

> n=4 的最終成功率（~29%）比 TD(0)（~25%）略高，比蒙特卡羅（~40%）低——符合「n 越大、估計越準但波動越大」的預期。

## 9. 與其他方法比較

| 演算法 | 更新時機 | 前看幾步 | 優點 |
|:---|:---:|:---:|:---|
| TD(0) / SARSA | 每步 | 1 步 | 最快反應 |
| N-步 TD（n=4）| 每 4 步 | 4 步 | 平衡速度與準確 |
| TD(λ) | 每步（跡傳播）| 加權平均 | 更靈活，不需固定 n |
| 蒙特卡羅 | 每集結束 | 整集 | 最準，但需等整集 |

**N-步 TD vs TD(λ)**：
- N-步 TD 用固定的 n，簡單直觀，容易理解和除錯
- TD(λ) 把所有 n 的回報加權混合（指數衰減），更靈活但實作複雜
- 兩者最終效果相近，N-步 TD 的方式更容易擴充套件到現代深度 RL（如 A3C、PPO）
