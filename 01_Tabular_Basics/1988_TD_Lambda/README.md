# TD(λ) — 資格跡 (Eligibility Traces)

> 年代：1988｜方法：無模型、每步線上更新、λ 統一 TD(0) 與 MC｜環境：FrozenLake-v1

---

## 1. TD(λ) 是什麼？

想像你終於走出了 FrozenLake，成功抵達終點——但你花了二十步才到。

**MC 的做法**：等走完整集，回頭倒算每步的 G，一口氣更新所有走過的格子。缺點是必須等整集結束，一步都不能提前學。

**TD(0) 的做法**：每走一步就立刻更新，不等整集結束。但只更新「剛走的那一格」——其他走過的格子完全沒有得到這一步的訊號。

**TD(λ) 的做法**：每走一步就立刻更新，同時把這一步的訊號往回傳——走得越近的格子得到越多，走得越遠的格子得到越少。靠「資格跡」記錄哪些格子最近被走過、走了多久以前。

**λ 就是一個旋鈕：轉向 0 就是 TD(0)，轉向 1 就是 MC。**

| 比較專案 | MC（蒙特卡羅）| TD(0) | TD(λ) |
|:---|:---:|:---:|:---:|
| 需要走完整集？ | ✅ 必須 | ❌ 不需要 | ❌ 不需要 |
| 怎麼更新 | 倒算 G，整集才更新 | 每步更新，僅當前格 | 每步更新，整條路徑 |
| 信用如何分配 | 所有走過的格平等 | 只給剛走的格 | 越近越多，指數衰減 |
| 等待時間 | 整集結束後 | 每步立刻 | 每步立刻 |

## 2. FrozenLake 環境

4 × 4 的冰湖，共 16 個格子：

```
 S  F  F  F       0   1   2   3
 F  H  F  H  →   4  [5]  6  [7]
 F  F  F  H       8   9  10  [11]
 H  F  F  G      [12] 13  14   15
```

- **S（0）**：起點
- **F**：普通冰面，可行走
- **H（5, 7, 11, 12）**：洞口，掉進去就結束這集，獎勵 0
- **G（15）**：終點，抵達得 1 分

**冰面會滑！** 你選擇往「下」走，實際有：
- 1/3 機率真的往下
- 1/3 機率往右滑
- 1/3 機率往左滑

所以就算走對方向，也不保證走到想去的格子。

## 3. 三個核心概念

### 3-1. 集數（Episode）

從起點出發，走到終點或掉進洞口，這一整段路叫做「一集」。  
TD(λ) **不需要**走完整集，每走一步就立刻更新 Q 值。

```
一集範例：
起點(0) → 格子4 → 格子8 → 格子9 → 格子13 → 格子14 → 終點(15) ✓
```

### 3-2. TD 誤差 δ（Temporal Difference Error）

每走一步，立刻比較「這一步的即時獎勵＋對未來的預期」與「目前的估計」之間的差距：

```
δ = r + γ × Q(s′, a′) − Q(s, a)      （集數未結束）
δ = r − Q(s, a)                        （集數結束，無下一步）

  r       ：這一步的即時獎勵
  γ = 0.99：折扣因子（未來獎勵打 1% 折）
  Q(s′,a′)：下一格用當前策略走的預期價值
  Q(s, a) ：目前格子的估計價值
```

**為什麼不等整集結束就更新？**  
MC 等整集走完才能算 G，如果一集走了一千步，每次都等到最後才學——很慢。  
TD 誤差 δ 只需要「下一格的估計值」就能馬上更新，不需要等終點。這個用估計值來引導另一個估計值的做法叫**引導（Bootstrapping）**。

### 3-3. 資格跡 E(s,a)（Eligibility Trace）

「這個格子-動作組合最近有被走過嗎？走了多久以前？」

資格跡是一個和 Q 表一樣大的矩陣，記錄每個（格子, 動作）組合的「新鮮度」：

```
每次訪問 (s, a)：E(s, :) ← 0          （替換式：清除 s 的所有跡）
                 E(s, a) ← 1          （再把剛訪問的重置為 1）
每過一步：       E(s, a) ← γλ × E(s,a) （所有跡以 γλ 的比例衰減）
```

當 TD 誤差 δ 出現，它按照跡的大小同時更新所有帶跡的格子：

```
Q(s,a) ← Q(s,a) + α × δ × E(s,a)   （對所有 s, a 同時進行）
```

**為什麼要衰減？**  
越久以前走的格子，對「現在這一步的結果」貢獻越小。衰減讓信用按時間距離自動分配——這正是「資格跡」名稱的由來：一個格子「有資格」接受多少信用，由它的跡值決定。

## 4. ε-greedy 探索策略

**問題**：每次都選 Q 最高的動作，可能永遠不去探索其他路線，錯過更好的走法。

**解法**：大多數時候選最好的，偶爾隨機探索。

```
以機率 ε   → 隨機選一個動作（探索）
以機率 1-ε → 選 Q 最高的動作（利用）
```

**ε 衰減**：訓練初期 ε = 1.0（幾乎全部隨機探索），隨著訓練進行逐漸降低，到 ε = 0.01（99% 按最佳策略走）。

| 訓練階段 | ε 約值 | 行為 |
|:---:|:---:|:---|
| 剛開始 | 1.00 | 完全隨機，像個迷路的觀光客 |
| 中間段 | 0.14 | 主要按最佳走法，偶爾隨機探索 |
| 末期 | 0.01 | 幾乎全按最佳走法，很少嘗試 |

## 5. TD(λ) SARSA 演演算法步驟

```
重複以下流程 N 次（N = 20,000 集）：

步驟一：重置資格跡 E ← 0，觀察初始格子 s，用 ε-greedy 選動作 a

步驟二：走一步，觀察 r, s′，用 ε-greedy 選下一動作 a′

步驟三：計算 TD 誤差
         δ = r + γ × Q(s′, a′) − Q(s, a)
         （集數結束時：δ = r − Q(s, a)）

步驟四：更新資格跡（替換式）
         E(s, :) ← 0
         E(s, a) ← 1

步驟五：按跡更新所有 Q 值，然後衰減跡
         Q(s,a) ← Q(s,a) + α × δ × E(s,a)   對所有 (s,a)
         E(s,a) ← γ × λ × E(s,a)             對所有 (s,a)

步驟六：s ← s′，a ← a′；若集數未結束，回到步驟二

步驟七：ε 稍微衰減（× 0.9998）

步驟八：回到步驟一，繼續下一集
```

**為什麼叫 SARSA(λ)？**  
每一步都需要「目前的格子 S、動作 A、獎勵 R、下一格 S′、下一動作 A′」這五個值，加上 λ 控制跡的衰減速度，所以叫 SARSA(λ)。

**λ 的意義：**

| λ 值 | 對應方法 | 衰減係數 γλ | 更新範圍 | 偏差 | 變異數 |
|:---:|:---:|:---:|:---|:---:|:---:|
| 0.0 | TD(0) | 0.000 | 僅當前步驟 | 高 | 低 |
| 0.5 | 中間地帶 | 0.495 | 近幾步 | 折衷 | 折衷 |
| 0.9 | 接近 MC | 0.891 | 大部分歷史 | 低 | 高 |
| 1.0 | MC 等效 | 0.990 | 整條軌跡 | 無偏差 | 最高 |

## 7. 訓練結果

四個 λ 值訓練 20,000 集的典型進度：

| 集數 | λ=0.0 成功率 | λ=0.5 成功率 | λ=0.9 成功率 | λ=1.0 成功率 |
|:---:|:---:|:---:|:---:|:---:|
| 5,000 | ~4% | ~4% | ~4% | ~2% |
| 10,000 | ~18% | ~18% | ~12% | ~7% |
| 15,000 | ~39% | ~39% | ~25% | ~16% |
| 20,000 | ~58% | ~58% | ~43% | ~31% |

**為什麼 λ=0 在短期訓練（20,000 集）中反而最快？**  
λ 越大，跡傳播範圍越廣——但同時也把不準確的 Q 值估計散播給更多格子，引入更高的**變異數**。訓練初期 Q 值幾乎都是零，廣泛傳播等於把雜訊放大，反而讓 Q 值震盪不穩。λ=0.0 每步只更新一格，更新雖窄但精準，20,000 集的短期訓練中收斂反而更快。

**為什麼最高只有 58% 左右？**  
FrozenLake 的冰面很滑，就算走最佳路線，仍然有隨機滑動。理論上最佳策略的成功率約為 74%——需要更多集數才能接近。

## 8. TD(λ) vs MC 比較

| 比較專案 | MC（蒙特卡羅）| TD(λ) |
|:---|:---|:---|
| **需要完整集數** | ✅ 必須等整集結束 | ❌ 每步立刻更新 |
| **學習來源** | 整集的實際回報 G | TD 誤差 δ（引導） |
| **信用分配方式** | 倒算 G，整集等比 | 跡矩陣 E，指數衰減 |
| **偏差** | 無偏差 | λ 越小偏差越高 |
| **變異數** | 最高（整集隨機的總和）| λ 越小變異數越低 |
| **無終點的環境** | ❌ 無法使用 | ✅ 可以使用 |
| **超引數** | γ, ε | γ, α, λ, ε |

**TD(λ) 的最大優勢**：不需要等整集結束，每走一步就能學習，適用於沒有明確終點的連續任務。透過 λ 這個旋鈕，可以在「快速收斂（低 λ）」和「低偏差（高 λ）」之間按問題需求自由調節。

## 9. TD(λ) 的限制（為什麼接下來需要函式近似？）

TD(λ) 有兩個根本弱點：

**弱點一：表格無法擴充**  
Q 表和 E 表的大小都等於「狀態數 × 動作數」。FrozenLake 只有 16×4=64 個值，但真實世界的狀態可能是幾億甚至連續的（例如機器人的關節角度、遊戲的畫素畫面）——根本存不進記憶體，也算不完。

**弱點二：λ 需要額外調參**  
多了一個超引數 λ。不同環境、不同集數長度、不同獎勵密度，最佳 λ 都不一樣。λ 選錯比不用跡還糟——λ 太大在短期訓練中反而收斂更慢（見第 7 節）。

**函式近似（Function Approximation）**的解法：  
不用表格存 Q(s,a)，改用神經網路來近似。TD(λ) 的 δ 訊號照樣可以拿來訓練網路——這正是後來 DQN 的核心想法：用 TD 誤差當損失函式，用卷積網路處理畫素輸入，表格的限制就此突破。

## 10. 結語

TD(λ) 是 1988 年 Sutton 提出的橋接框架，核心洞察只有一句話：**資格跡是一種高效的信用分配機制**。

它不需要儲存整條軌跡（像 MC 那樣），也不需要知道環境的轉移機率（像 DP 那樣），只需要一個和 Q 表一樣大的跡矩陣 E，就能在每步更新時把「現在的獎勵訊號」自動地、按比例地傳回給所有最近走過的格子。

MC 證明瞭「不需要地圖也能學」；TD(0) 加上了「不需要等整集結束」；TD(λ) 則把兩者真正統一——用一個引數 λ 在偏差與變異數之間連續調節。這套思想後來直接催生了線性函式近似 TD、再到 DQN，最終成為現代深度強化學習的演算法核心。
