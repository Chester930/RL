# 掃地機器人 Q-Learning 實戰

這是一個用於理解 Q-Learning 核心更新公式的教學專案。

## 場景設定：4x4 迷宮
我們訓練一台掃地機器人（Agent），在一個 4x4 的房間內移動。
- **狀態 (States)**: 0 到 15（共 16 個格子）。
- **動作 (Actions)**: 0(上), 1(下), 2(左), 3(右)。
- **獎勵 (Rewards)**:
  - 找到充電座（目標）：`+10`
  - 踩到水坑（陷阱）：`-5`
  - 每移動一步（日常消耗）：`-0.1` （促使機器人尋找最短路徑）

## Q-Learning 核心公式對照

Q-Learning 的精髓在於更新知識庫（Q-Table）。公式如下：
$$Q(s, a) \leftarrow Q(s, a) + \alpha \cdot \delta$$

其中，$\delta$ 稱為 **時序差分誤差 (TD Error)**：
$$\delta = \text{TD Target} - Q(s, a)$$

而 **預期回報 (TD Target)** 是：
$$\text{TD Target} = r + \gamma \cdot \max_{a'} Q(s', a')$$

### 在程式碼中的對應 (`agent.py`)
- $r$: `reward` (環境給予的回饋)
- $\gamma$: `self.gamma` (折扣因子，決定對未來回報的重視程度)
- $\max_{a'} Q(s', a')$: `self.Q[next_state].max()` (下一個狀態中，所有可能動作的最高 Q 值)
- $Q(s, a)$: `self.Q[state, action]` (目前狀態與動作的 Q 值)
- $\alpha$: `self.alpha` (學習率，決定每次更新的步伐大小)

透過本專案的 `agent.py` 中的註解，你可以清楚看到數學公式是如何 1:1 轉換為 Python 程式碼的。

## 如何解讀訓練日誌 (Q-Table)

在 `training_log.txt` 中，你會看到矩陣形狀的輸出，它是一個 `16 x 4` 的矩陣：
- **行 (Row，直向 0~15)**：代表機器人所在的**狀態 (State)**。第 0 行對應起點，第 15 行對應充電座。
- **列 (Column，橫向 0~3)**：代表可以採取的**動作 (Action)**，依序為 `[上, 下, 左, 右]`。
- **裡面的數字**：就是機器人認為在「這個狀態下，採取這個動作」未來能獲得的**預期總回報 (Q-Value)**。

**舉例說明：**
假設你在 `training_log.txt` 看到以下這列 (代表狀態 0 的 Q 值)：
```text
 [ 0.00 -0.50  0.00  0.00]
```
這代表：
- 在狀態 0 往「上」(第0欄)：預期回報 0.00（撞牆，原地不動）
- 在狀態 0 往「下」(第1欄)：預期回報 -0.50
- 在狀態 0 往「左」(第2欄)：預期回報 0.00（撞牆，原地不動）
- 在狀態 0 往「右」(第3欄)：預期回報 0.00

隨著訓練次數增加，這些數字會往上傳遞，機器人最終會選擇每一列中**數字最大**的那個動作作為最佳策略 (Greedy Policy)。
