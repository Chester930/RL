# Q-Learning（Q 學習）

> 年代：1989｜方法：無模型、異策略、每步線上更新｜環境：FrozenLake-v1

---

## 1. Q-Learning 是什麼？

想像你在 FrozenLake 裡邊走邊學，但你有一個特別的能力：**每次更新時，你假設自己下一步一定會做出最聰明的選擇**——不管你實際走了哪一步。

**MC 的做法**：走完整集，再回頭更新。學的是「按現在策略走，平均能拿多少分」。

**TD(0)/SARSA 的做法**：每步立刻更新，但目標是「下一步實際選的那個動作的 Q 值」——學的是當前策略的價值。

**Q-Learning 的做法**：每步立刻更新，目標是「下一格所有動作中最大的 Q 值」——不管你的探索策略實際選了哪一步，學的直接是**最優策略的價值 Q\***。

**這叫「異策略（Off-policy）」：用一個策略探索，同時學習另一個（最優）策略。**

| 比較專案 | MC | SARSA（同策略）| Q-Learning（異策略）|
|:---|:---:|:---:|:---:|
| 需要走完整集？ | ✅ 必須 | ❌ 不需要 | ❌ 不需要 |
| 更新目標 | 實際回報 G | Q(s′, 實際選的 a′) | max Q(s′, 所有 a′) |
| 學習的策略 | 當前策略 | 當前策略 | 最優策略 Q* |
| 探索會影響 Q？ | 會 | 會 | **不會** |

## 2. FrozenLake 環境

4 × 4 的冰湖，共 16 個格子：

```
 S  F  F  F       0   1   2   3
 F  H  F  H  →   4  [5]  6  [7]
 F  F  F  H       8   9  10  [11]
 H  F  F  G      [12] 13  14   15
```

- **S（0）**：起點
- **F**：普通冰面，可行走
- **H（5, 7, 11, 12）**：洞口，掉進去就結束這集，獎勵 0
- **G（15）**：終點，抵達得 1 分

**冰面會滑！** 你選擇往「下」走，實際有：
- 1/3 機率真的往下
- 1/3 機率往右滑
- 1/3 機率往左滑

所以就算走對方向，也不保證走到想去的格子。

## 3. 三個核心概念

### 3-1. 集數（Episode）

從起點出發，走到終點或掉進洞口，這一整段路叫做「一集」。  
Q-Learning **不需要**走完整集，每走一步就立刻更新 Q 值。

```
一集範例：
起點(0) → 格子4 → 格子8 → 格子9 → 格子13 → 格子14 → 終點(15) ✓
```

### 3-2. 異策略目標（Off-policy Target）

Q-Learning 每一步的更新目標是：

```
目標 = r + γ × max_{a′} Q(s′, a′)

  r                  ：這一步的即時獎勵
  γ = 0.99           ：折扣因子
  max Q(s′, a′)      ：下一格所有動作的 Q 值中，取最大的那個
```

**為什麼取 max 而不是實際走的那一步？**  
Q-Learning 想學的是「最優」的 Q 值，也就是貝爾曼最優方程式（Bellman Optimality Equation）的解：

```
Q*(s, a) = r + γ × max_{a′} Q*(s′, a′)
```

用 max 作為目標，保證每次更新都在往最優 Q* 靠近——無論行為策略是否隨機探索，學到的 Q 值始終對應最優策略。

### 3-3. 動作價值 Q(s, a)

「在格子 s 採取動作 a，**之後每步都做最優選擇**，預期能得到多少折扣後的總獎勵。」

```
Q(s, a) ← Q(s, a) + α × δ

  δ = 目標 − Q(s, a)   ← TD 誤差
  α = 0.1              ← 學習率

  δ > 0 → 目標比目前估計高，調高 Q(s,a)
  δ < 0 → 目標比目前估計低，調低 Q(s,a)
  δ = 0 → 估計已精準，不動
```

**增量更新的意義**：

```
α=0.1 時：每次更新只移動 10% 的距離
         → 不會因為一次隨機結果就大幅改變估計
         → 多次更新後，Q 值慢慢收斂到真實的 Q*
```

## 4. ε-greedy 探索策略

**問題**：每次都選 Q 最高的動作，可能永遠不去探索其他路線，錯過更好的走法。

**解法**：大多數時候選最好的，偶爾隨機探索。

```
以機率 ε   → 隨機選一個動作（探索）
以機率 1-ε → 選 Q 最高的動作（利用）
```

**ε 衰減**：訓練初期 ε = 1.0（幾乎全部隨機探索），隨著訓練進行逐漸降低，到 ε = 0.01（99% 按最佳策略走）。

| 訓練階段 | ε 約值 | 行為 |
|:---:|:---:|:---|
| 剛開始 | 1.00 | 完全隨機，像個迷路的觀光客 |
| 中間段 | 0.14 | 主要按最佳走法，偶爾隨機探索 |
| 末期 | 0.01 | 幾乎全按最佳走法，很少嘗試 |

**Q-Learning 的特別之處**：即使 ε=1.0 完全隨機探索，Q 值依然會往最優 Q* 收斂，因為更新目標用的是 max Q，不受探索行為的幹擾。

## 5. Q-Learning 演演算法步驟

```
重複以下流程 N 次（N = 20,000 集）：

步驟一：觀察初始格子 s，用 ε-greedy 選動作 a

步驟二：走一步，觀察 r 和下一格 s′

步驟三：計算異策略 TD 目標
         目標 = r + γ × max_{a′} Q(s′, a′)
         （集數結束時：目標 = r）

步驟四：計算 TD 誤差
         δ = 目標 − Q(s, a)

步驟五：更新 Q(s, a)
         Q(s, a) ← Q(s, a) + α × δ

步驟六：s ← s′；若集數未結束，回到步驟一

步驟七：ε 稍微衰減（× 0.9998）

步驟八：回到步驟一，繼續下一集
```

**為什麼只更新 Q(s, a)，不更新其他格子？**  
Q-Learning 每步只修正「這一步剛走過的那個 (格子, 動作)」。沒有資格跡（Eligibility Trace），不會回傳給更早的格子。信用是靠「下次走到這格的相鄰格子時，把剛更新的 Q 值當成 Bootstrap 目標」這樣一步一步往前傳的——比 TD(λ) 慢，但結構更簡單。

**Q-Learning vs SARSA 的關鍵差異：**

```
Q-Learning：目標 = r + γ × max Q(s′, a′)   ← 異策略，下一步取最大
SARSA      ：目標 = r + γ × Q(s′, a′)       ← 同策略，下一步取實際選的
```

## 7. 訓練結果

訓練 20,000 集的典型進度：

| 集數 | 平均成功率 | ε 約值 | 說明 |
|:---:|:---:|:---:|:---|
| 5,000 | ~4% | 0.37 | 幾乎全探索，很少成功 |
| 10,000 | ~20% | 0.14 | 開始學到靠近終點的路線 |
| 15,000 | ~40% | 0.05 | 進步明顯，策略趨於穩定 |
| 20,000 | ~58% | 0.01 | 最終策略趨於穩定 |

**為什麼最高只有 58% 左右？**  
FrozenLake 的冰面很滑，就算走最佳路線，仍然有隨機滑動。理論上最佳策略的成功率約為 74%——需要更多集數和更細緻的探索才能接近。

## 8. Q-Learning vs SARSA 比較

| 比較專案 | Q-Learning（異策略）| SARSA（同策略）|
|:---|:---|:---|
| **更新目標** | r + γ × **max** Q(s′, a′) | r + γ × Q(s′, **實際選的** a′) |
| **學習的策略** | 最優策略 Q* | 當前 ε-greedy 策略 |
| **探索影響** | ε 不影響學到的 Q 值 | ε 越大，Q 越保守 |
| **懸崖邊** | 走捷徑，可能掉下去 | 繞遠路，更安全 |
| **收斂目標** | Q*（最優） | Q^π（當前策略的 Q）|
| **適用場景** | 離線學習、重放緩衝 | 線上部署、安全敏感 |

**Q-Learning 的最大優勢**：學習與探索完全分離。你可以用完全隨機的策略收集資料，Q-Learning 照樣能從中學出最優策略——這讓它天然適合「先收集資料、再離線訓練」的場景，也是後來 DQN 使用 Replay Buffer 的理論基礎。

## 9. Q-Learning 的限制（為什麼接下來需要函式近似？）

Q-Learning 有兩個根本弱點：

**弱點一：表格無法擴充**  
Q 表大小 = 狀態數 × 動作數。FrozenLake 只有 16×4=64 個值，但真實世界的狀態可能是幾億個（棋盤）甚至連續的（機器人關節角度、遊戲畫素畫面）——根本存不進記憶體。

**弱點二：信用傳播很慢**  
沒有資格跡，每次只更新一個 (s, a)。終點附近的格子需要很多集才能讓信用一步一步傳回到起點附近。FrozenLake 簡單還好，獎勵稀疏的複雜環境需要極大量集數。

**函式近似（Function Approximation）**的解法：  
不用表格存 Q(s,a)，改用神經網路來近似。Q-Learning 的更新規則完全不變——只是把 `Q[state, action]` 換成神經網路的輸出。這就是 **DQN（Deep Q-Network）** 的核心：Q-Learning + 卷積神經網路 + Replay Buffer，讓 Q-Learning 從 16 格棋盤擴充套件到 Atari 遊戲畫面。

## 10. 結語

Q-Learning 是 1989 年 Watkins 在博士論文中提出的演演算法，核心洞察只有一句話：**用 max 當目標，就能直接學最優策略**。

它把 TD(0) 的「每步更新」效率，和 MC 的「不需要地圖」靈活性結合在一起，再加上「異策略」這一個關鍵突破——把探索行為和學習目標分離。這樣的設計讓收集資料的策略和正在學習的策略可以完全不同，為後來的 Replay Buffer、Offline RL 等技術埋下了種子。

從 16 格的 FrozenLake 到 2013 年 DeepMind 用 DQN 打 Atari，核心的 Q-Learning 更新公式一字未變。理解 Q-Learning，就理解了現代深度強化學習最重要的根基之一。
